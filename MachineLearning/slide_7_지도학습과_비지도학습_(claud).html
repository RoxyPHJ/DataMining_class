01

지도학습 vs 비지도학습 — 학습 패러다임

핵심 차이: 지도학습은 '정답이 있는 학습', 비지도학습은 '정답 없이 패턴을 찾는 학습'이다.
✓ 지도학습 (Supervised Learning)
학습 목표
입력 x와 레이블 y의 쌍으로 학습
P(y|x) 또는 f(x) = y를 추정
핵심 질문
"이 데이터가 어느 클래스인가?" / "출력 값이 얼마인가?"
학습 데이터
{ (x₁,y₁), (x₂,y₂), … , (xₙ,yₙ) }
레이블이 반드시 필요
대표 알고리즘
KNN 로지스틱 회귀 SVM MLP
◎ 비지도학습 (Unsupervised Learning)
학습 목표
레이블 없이 데이터 자체의 구조 발견
P(x) 또는 잠재 구조를 추정
핵심 질문
"데이터가 어떻게 그룹화되는가?" / "데이터의 핵심 구조는 무엇인가?"
학습 데이터
{ x₁, x₂, … , xₙ }
레이블 불필요
대표 알고리즘
K-means GMM PCA Autoencoder
02

KNN (K-Nearest Neighbors)

"새로운 데이터 포인트 x가 주어졌을 때, 훈련 데이터 중 가장 가까운 K개의 이웃을 찾아 다수결(분류) 또는 평균(회귀)으로 예측한다."
핵심 아이디어
명시적 모델 없음
파라미터를 학습하지 않는다
→ Instance-based / Lazy Learning
유사성 기반
비슷한 입력은 비슷한 출력을 가진다는 가정
K는 하이퍼파라미터
K 작음 → 복잡한 경계
K 큼 → 부드러운 경계
KNN 직관적 그림 — K=3 예시
거리 측도 (Distance Metric)
'가장 가깝다'는 것을 어떻게 정의하는가? — 거리 함수의 선택이 KNN 성능을 좌우한다.
유클리드 거리 (L2)
d(x,z) = √(Σᵢ(xᵢ−zᵢ)²)
두 점 사이의 직선 거리
→ 연속형 데이터에 가장 많이 사용
→ 모든 특성의 스케일이 같아야 함
→ 정규화 필수
맨해튼 거리 (L1)
d(x,z) = Σᵢ|xᵢ−zᵢ|
격자 이동 거리 (택시 거리)
→ 이상치(outlier)에 더 강건
→ 고차원에서 유클리드보다 안정적
민코우스키 거리 (Lp)
d(x,z) = (Σᵢ|xᵢ−zᵢ|ᵖ)^(1/p)
유클리드(p=2)와 맨해튼(p=1)의 일반화
→ p→∞ : 체비쇼프 거리
   (최대 성분 차이)
⚠ 스케일 정규화 필수: 키(cm)와 체중(kg)을 그대로 사용하면 단위 차이로 거리가 왜곡된다. 반드시 정규화(Normalization) 또는 표준화(Standardization)를 먼저 수행해야 한다.
분류 알고리즘 단계
①
훈련 (Train)
훈련 데이터 X = {(x₁,y₁), … , (xₙ,yₙ)}를 메모리에 저장
→ 별도의 파라미터 학습 없음 Lazy Learning
②
거리 계산 (Predict)
새로운 데이터 xₜₑₛₜ와 모든 훈련 데이터 간의 거리 d(xₜₑₛₜ, xᵢ)를 계산
③
이웃 선택
거리 기준으로 정렬 → 가장 가까운 K개의 이웃 { xᵢ₁, … , xᵢₖ } 선택
④
예측 — 분류 (다수결 투표)
이웃 K개에서 가장 많은 클래스로 분류
④'
예측 — 회귀 (평균)
이웃 K개의 출력값 평균
ŷ = (1/K) Σⱼ yᵢⱼ
시간복잡도: 훈련 O(1) — 예측 O(n·d)  |  공간복잡도: O(n·d) → 데이터가 클수록 예측이 느려짐
수치 예시 (K=3, 2차원)
설정: 훈련 데이터 5개, 분류 대상 xₜₑₛₜ = (5,4), K = 3, 유클리드 거리 사용
데이터x₁x₂레이블d(xₜₑₛₜ, xᵢ)이웃?
x₁34A √((5−3)²+(4−4)²) = 2.00 ✓ 1위
x₂62B √((5−6)²+(4−2)²) = 2.24 ✓ 2위
x₃46A √((5−4)²+(4−6)²) = 2.24 ✓ 3위
x₄83B √((5−8)²+(4−3)²) = 3.16 ✗
x₅12A √((5−1)²+(4−2)²) = 4.47 ✗
다수결 투표 (K=3)
K=3 이웃: x₁(A), x₂(B), x₃(A)
클래스 A: 2표 / 클래스 B: 1표
→ 다수결 → 클래스 A
최종 예측
ŷ = 클래스 A
xₜₑₛₜ = (5,4)는 클래스 A로 분류된다.
K 값의 영향: K=1이면 x₁(A)만으로 A. K=5면 A:3표, B:2표 → A.
K가 너무 크면 경계가 단순해져 과소적합(underfitting) 발생 가능.
K 값의 영향과 Bias-Variance Tradeoff
K가 결정 경계의 복잡도를 제어한다.
K = 1

가장 가까운 1개만 참조
→ 훈련 오류 = 0
→ 결정 경계 매우 복잡
→ 과적합(Overfitting) 위험

낮은 편향 / 높은 분산
K = n

모든 훈련 데이터가 이웃
→ 항상 가장 많은 클래스로 분류
→ 결정 경계 매우 단순
→ 과소적합(Underfitting)

높은 편향 / 낮은 분산
최적 K

교차 검증(Cross-Validation)으로 선택
→ 편향과 분산의 균형점
→ 홀수 K 사용 (동점 방지)
→ 시작점: K = √n

편향: 균형 / 분산: 균형
K 선택 방법 — k-fold 교차검증:
훈련 데이터를 k개 폴드로 분할 → 각 폴드를 순서대로 검증 데이터로 사용 → 검증 오류가 최소인 K를 선택
→ 홀수 K 권장, 일반적으로 K = √n 을 시작점으로 설정
차원의 저주 (Curse of Dimensionality)
⚠ KNN의 가장 치명적인 약점 — 차원이 높아질수록 거리 개념이 무의미해진다.
직관적 이해
거리의 경우 모든 특성들이 "똑같이 중요하게" 계산된다.
d(x,z) = √(Σᵢ(xᵢ−zᵢ)²)
암 분류 예시 — 특성과 예측 연관성
특성변수예측 연관성
종양 크기x₁매우 높음
세포 모양 불규칙도x₂매우 높음
환자 혈액형x₃거의 없음
환자 생년월일x₄거의 없음
병원 방문 요일x₅거의 없음
1. 거리 집중 현상
모든 점들이 거의 같은 거리에 위치
→ '가장 가까운 이웃' 구분 불가
2. 노이즈의 증가
변수가 늘어날수록 예측과 무관한 노이즈 변수 증가
→ 혈액형이 같은 환자가 종양 크기 유사 환자보다 더 가까운 이웃이 될 수 있음
3. 계산 비용 증가
거리 계산 비용 O(n·d)
→ d가 크면 예측이 매우 느려짐
해결 방향 → PCA / 차원 축소: 특성 선택(Feature Selection) 또는 PCA로 차원을 줄인 뒤 KNN을 적용하면 성능이 크게 향상된다.
장단점 및 실용적 고려사항
✓ 장점 (Strengths)
구현이 단순하다 — 핵심 가정이 없음
훈련이 필요 없다 (Lazy Learning)
다중 클래스 분류를 자연스럽게 지원
비선형 결정 경계를 자동으로 표현
새로운 훈련 데이터 추가가 쉽다
✗ 단점 (Weaknesses)
예측 시간 O(n·d) — 대규모 데이터에 느림
차원의 저주 — 고차원에서 성능 저하
메모리에 전체 훈련 데이터 저장 필요
스케일에 민감 — 반드시 정규화 필요
특성 중요도를 알 수 없음 (해석 불가)
실용적 체크리스트
전처리
Min-Max 정규화 또는 Z-score 표준화 수행
K 선택
k-fold 교차검증으로 최적 K 탐색 (홀수 권장)
거리 선택
연속형 → 유클리드 / 이산형 → 맨해튼 / 문자열 → 편집 거리
속도 개선
KD-Tree 또는 Ball-Tree 자료구조로 이웃 탐색 가속화
03

K-means 클러스터링

"n개의 데이터를 K개의 클러스터로 분할하되, 각 데이터는 가장 가까운 클러스터 중심(centroid)에 배정되고, 클러스터 내 분산이 최소가 되도록 중심을 반복 갱신한다."
목적함수 — Inertia / WCSS
J = Σₖ Σ(xᵢ∈Cₖ) ||xᵢ − μₖ||²
→ argmin(C₁,…,Cₖ, μ₁,…,μₖ) J
μₖ : 클러스터 k의 중심 (centroid)
Cₖ : 클러스터 k에 속한 데이터 집합
KNN과의 핵심 차이
KNN  레이블 있음 (지도) / 기존 레이블 기반 분류
K-means  레이블 없음 (비지도) / 새로운 그룹 구조 발견
클러스터링 결과 예시
알고리즘 단계 — Lloyd's Algorithm
①
초기화 (Initialize)
K개의 클러스터 중심 μ₁, μ₂, ..., μₖ를 임의로 선택
(랜덤 샘플링 또는 K-means++ 방식 사용)
②
할당 단계 (Assignment Step)
각 데이터 xᵢ를 가장 가까운 클러스터 중심에 할당
cᵢ = argmin_k ||xᵢ − μₖ||²
→ 각 데이터는 정확히 하나의 클러스터에 속함
③
갱신 단계 (Update Step)
각 클러스터의 새로운 중심을 해당 클러스터 데이터의 평균으로 갱신
μₖ = (1/|Cₖ|) Σ(xᵢ∈Cₖ) xᵢ
④
반복 (Repeat)
② → ③ 를 클러스터 할당이 변하지 않을 때까지 반복
→ 수렴 조건: 모든 cᵢ가 이전 반복과 동일할 때
수렴 보장: 각 반복에서 목적함수 J는 단조 감소 → 반드시 유한 번 반복 후 수렴
단, 전역 최솟값 보장은 안 됨
수치 예시 (K=2, 1차원)
설정: 데이터 = {1, 2, 8, 9, 10}, K=2, 초기 중심 μ₁=1, μ₂=9 (1차원, 유클리드 거리)
1차원 데이터 클러스터링 시각화
반복 1 — 할당 단계
데이터d(x, μ₁=1)d(x, μ₂=9)배정
x=1 |1−1| = 0 |1−9| = 8 C₁
x=2 |2−1| = 1 |2−9| = 7 C₁
x=8 |8−1| = 7 |8−9| = 1 C₂
x=9 |9−1| = 8 |9−9| = 0 C₂
x=10 |10−1| = 9 |10−9| = 1 C₂
반복 1 — 갱신 단계
새 μ₁ = (1+2)/2 = 1.5
새 μ₂ = (8+9+10)/3 = 9.0
반복 2 → 수렴!
C₁={1,2}, 배정 동일 → μ₁=1.5 (불변)
C₂={8,9,10}, 배정 동일 → μ₂=9.0 (불변)
→ 수렴! (2회 반복)
J = 0.5+0.5+1+0+1 = 3.0
수렴 보장 및 한계
수렴 증명 (직관)
1
할당 단계
각 xᵢ를 가장 가까운 중심에 배정 → 거리 합 J가 감소하거나 같아짐 (J_new ≤ J_old)
2
갱신 단계
클러스터 중심을 평균으로 갱신 → 평균은 제곱 오차를 최소화하는 점 → J가 감소하거나 같아짐
3
유한성
클러스터 할당의 경우의 수는 유한 (Kⁿ 개) → 단조 감소 + 유한 → 반드시 수렴
주의: 전역 최솟값 비보장
초기 중심값에 따라 서로 다른 지역 최솟값(local minimum)에 수렴할 수 있다.
→ 해결책: 여러 번 랜덤 초기화 후 가장 작은 J 값의 결과 선택 (n_init 파라미터)
결론: K-means는 수렴이 보장되지만 전역 최솟값 보장은 없다. 실전에서는 여러 번 실행 후 가장 좋은 결과를 사용한다.
초기화 문제와 K-means++
랜덤 초기화의 문제
초기 중심이 같은 클러스터 내에 여러 개 배치되면
→ 빈 클러스터 또는 나쁜 지역 최솟값 발생
→ 실제 클러스터 구조를 제대로 못 찾음
K-means++ 알고리즘
초기 중심들이 서로 멀리 떨어지도록 확률적 선택
→ 이미 선택된 중심들과 거리가 먼 점을 더 높은 확률로 다음 중심으로 선택
K-means++ 초기화 절차
1
첫 번째 중심 μ₁을 데이터에서 균일 랜덤 선택
2
각 데이터 xᵢ에 대해 이미 선택된 중심들까지의 최소 거리² 계산
D(xᵢ)² = min_k ||xᵢ − μₖ||²
3
D(xᵢ)²에 비례한 확률로 다음 중심 선택
P(xᵢ 선택) ∝ D(xᵢ)²
4
K개의 중심이 선택될 때까지 2~3 반복
5
이후 표준 K-means 알고리즘 실행
K-means++는 초기화만 바꿨을 뿐이지만 수렴 속도와 최종 품질을 크게 개선한다. scikit-learn 기본값으로 사용됨.
K 개수 선택 — Elbow Method
Elbow Method (팔꿈치 방법)
핵심 아이디어:
K를 늘릴수록 J (inertia)는 감소한다.
K=n이면 J=0 (자기 자신이 중심)
→ 단순히 J가 작은 K를 고르면 안 됨
Elbow 기준:
J 감소율이 급격히 줄어드는 'elbow' 지점
→ 추가 클러스터의 이득이 크지 않은 K 선택
항상 명확한 elbow가 보이지 않을 수 있다. 이 경우 실루엣 계수(Silhouette Score) 등 다른 지표를 병행한다.
한계 및 주의사항
①
K를 사전에 지정해야 한다
클러스터 개수를 미리 알아야 함
→ Elbow method, 실루엣 점수 등으로 탐색 필요
②
구형(원형) 클러스터 가정
유클리드 거리 기반 → 늘어진 타원형이나 복잡한 모양의 클러스터는 잘 포착 못 함
→ DBSCAN, GMM 등 대안 고려
③
이상치(Outlier)에 민감
중심 = 평균이므로 이상치가 중심을 크게 왜곡
→ K-medoids (중앙값 기반) 사용 검토
④
스케일에 민감
거리 기반 → 특성 스케일이 다르면 특정 특성이 지배
→ 표준화 (Z-score) 필수
⑤
이진 / 범주형 데이터 부적합
유클리드 거리가 의미 없음
→ K-modes (범주형) 또는 Gower 거리 사용
K-means는 단순하고 빠르지만 이러한 한계를 이해하고 데이터 특성에 맞는 알고리즘을 선택해야 한다.

요약 (Summary)

지도 vs
비지도
지도학습은 레이블로 f(x)=y를 학습. 비지도학습은 레이블 없이 P(x)의 구조를 발견.
KNN
핵심
K개 이웃의 다수결(분류) 또는 평균(회귀). 모델 학습 없는 Lazy Learning. 거리 정규화 필수.
KNN
주의
K 선택은 교차검증. 고차원에서 차원의 저주 발생 → PCA로 선처리 필요.
K-means
핵심
K개 클러스터, 반복 할당-갱신. 목적함수 J = Σ||xᵢ−μₖ||². 수렴은 보장, 전역최솟값은 미보장.
K-means
주의
K-means++ 초기화 권장. Elbow/Silhouette으로 K 선택. 구형 클러스터 가정 한계.
