단답형 ⭐ Slide 6
KNN 알고리즘에서 훈련(Train) 단계의 시간복잡도와 예측(Predict) 단계의 시간복잡도를 각각 Big-O 표기법으로 쓰시오. (n: 훈련 데이터 수, d: 특성 차원 수)
정답
훈련: O(1) — 파라미터를 학습하지 않고 데이터를 메모리에 저장만 하므로 훈련 비용이 없다.
예측: O(n·d) — 새 데이터와 n개의 훈련 데이터 각각에 대해 d차원 거리를 계산해야 하므로.
02
단답형 ⭐ Slide 5
KNN에서 사용하는 세 가지 거리 측도의 이름과 각각의 특징을 한 문장씩 서술하시오.
정답
① 유클리드 거리(L2): 두 점 사이의 직선 거리로, 연속형 데이터에 가장 많이 사용되며 스케일 정규화가 필요하다.
② 맨해튼 거리(L1): 격자 이동(택시) 거리로, 이상치에 강건하고 고차원에서 유클리드보다 안정적이다.
③ 민코우스키 거리(Lp): 유클리드(p=2)와 맨해튼(p=1)의 일반화 형태이며, p→∞이면 체비쇼프 거리(최대 성분 차이)가 된다.
03
단답형 ⭐⭐ Slide 12
K-means 알고리즘의 목적함수(WCSS / Inertia) J를 수식으로 쓰고, 각 기호의 의미를 설명하시오.
정답
J = Σₖ Σ(xᵢ∈Cₖ) ||xᵢ − μₖ||²
μₖ: 클러스터 k의 중심 (centroid) — 해당 클러스터에 속한 데이터들의 평균 벡터
Cₖ: 클러스터 k에 속한 데이터 집합
||xᵢ − μₖ||²: 각 데이터 포인트와 클러스터 중심 사이의 유클리드 거리의 제곱 (클러스터 내 분산)

단답형 [응용] ⭐⭐ Slide 7
아래 훈련 데이터와 쿼리 포인트가 주어졌을 때, K=3, 맨해튼 거리(L1)를 사용하여 xₜₑₛₜ의 클래스를 예측하시오.

훈련 데이터: x₁=(2,3,A), x₂=(5,1,B), x₃=(3,5,A), x₄=(7,2,B), x₅=(1,1,A)
쿼리: xₜₑₛₜ = (4, 3)
정답: 클래스 A
맨해튼 거리 계산 (L1):
d(xₜₑₛₜ, x₁) = |4−2|+|3−3| = 2+0 = 2 → 1위 ✓
d(xₜₑₛₜ, x₂) = |4−5|+|3−1| = 1+2 = 3 → 3위 ✓
d(xₜₑₛₜ, x₃) = |4−3|+|3−5| = 1+2 = 3 → 3위 ✓ (동거리, 둘 다 포함)
d(xₜₑₛₜ, x₄) = |4−7|+|3−2| = 3+1 = 4 → ✗
d(xₜₑₛₜ, x₅) = |4−1|+|3−1| = 3+2 = 5 → ✗

K=3 이웃: x₁(A), x₂(B), x₃(A) → A: 2표, B: 1표 → 클래스 A
변형 포인트
원본(Slide 7)과 동일한 KNN 풀이 구조이나, 거리 측도를 유클리드(L2)→맨해튼(L1)으로 변경하고 수치를 새로 설정. L1에서는 제곱근 없이 절댓값 합을 사용하는 점이 핵심 변형이다.

단답형 [응용] ⭐⭐⭐ Slide 13
K-means(K=2)를 1차원 데이터 {3, 4, 10, 11, 12}에 적용했더니 1회 반복 후 수렴했다.
이 결과가 나오려면 초기 중심 μ₁, μ₂를 각각 어떤 범위에서 선택했어야 하는가? (단, μ₁ < μ₂)
정답
최종 수렴 결과: C₁={3,4} (중심 3.5), C₂={10,11,12} (중심 11)

1회 반복에서 수렴하려면 초기 할당이 최종 결과와 동일해야 한다.
경계값은 두 클러스터 중간인 (4+10)/2 = 7이다.

따라서 초기 중심 조건:
μ₁ ∈ (−∞, 7) 범위의 임의 값 — 3, 4에 더 가까워야 함
μ₂ ∈ (7, +∞) 범위의 임의 값 — 10, 11, 12에 더 가까워야 함

더 정확히는: 5 이하에서 선택하면 x=4가 μ₁에 배정되고, 6 이상에서 선택하면 x=10이 μ₂에 배정됨이 보장된다.

단답형 [응용] ⭐⭐ Slide 13
K-means(K=2)를 데이터 {2, 3, 9, 10, 11}에 적용한다. 초기 중심은 μ₁=2, μ₂=10이다.
반복 1회 후의 새 중심 μ₁', μ₂'와 최종 목적함수 J를 구하시오.
정답
[할당 단계]
x=2: d(2,2)=0, d(2,10)=8 → C₁
x=3: d(3,2)=1, d(3,10)=7 → C₁
x=9: d(9,2)=7, d(9,10)=1 → C₂
x=10: d(10,2)=8, d(10,10)=0 → C₂
x=11: d(11,2)=9, d(11,10)=1 → C₂

[갱신 단계]
μ₁' = (2+3)/2 = 2.5
μ₂' = (9+10+11)/3 = 10.0

[반복 2 → 수렴 확인]
할당 경계 = (2.5+10)/2 = 6.25 → C₁={2,3}, C₂={9,10,11} 동일 → 수렴!

J = (2−2.5)²+(3−2.5)² + (9−10)²+(10−10)²+(11−10)²
   = 0.25+0.25 + 1+0+1 = 2.5
변형 포인트
원본 Slide 13({1,2,8,9,10}, μ₁=1, μ₂=9)에서 각 수치를 +1씩 이동. 풀이 구조는 동일하나 최종 J값이 3.0→2.5로 달라짐.

단답형 ⭐⭐ Slide 15
K-means++ 초기화에서, 두 번째 이후의 중심을 선택하는 확률 공식을 쓰고 그 의미를 설명하시오.
정답
P(xᵢ 선택) ∝ D(xᵢ)²
여기서 D(xᵢ)² = min_k ||xᵢ − μₖ||² (이미 선택된 중심들까지의 최소 거리의 제곱)

의미: 이미 선택된 중심들로부터 멀리 떨어진 점일수록 다음 중심으로 선택될 확률이 높아진다. 이를 통해 초기 중심들이 서로 멀리 분산되도록 유도하여, 나쁜 지역 최솟값(local minimum)을 회피한다.

코드완성형 ⭐⭐ Slide 12
아래는 K-means의 할당 단계(Assignment Step)를 구현한 코드이다. 빈칸 ①~②를 채우시오.
def assignment_step(X, centroids): """각 데이터를 가장 가까운 centroid에 배정""" labels = [] for x in X: distances = [ euclidean_distance(x, mu) for mu in centroids ] # ① 가장 가까운 centroid의 인덱스 선택 nearest = ① labels.append(nearest) return labels def update_step(X, labels, k): """갱신 단계: 각 클러스터의 평균으로 중심 갱신""" new_centroids = [] for cluster_id in range(k): members = [X[i] for i in range(len(X)) if labels[i] == cluster_id] # ② 클러스터 평균 계산 new_mu = ② new_centroids.append(new_mu) return new_centroids
정답
① distances.index(min(distances))
— 거리 리스트에서 최솟값의 인덱스를 반환 → argmin_k ||xᵢ − μₖ||² 구현

② [sum(col)/len(col) for col in zip(*members)]
— zip(*members)로 차원별로 묶고, 각 차원의 평균을 계산 → μₖ = (1/|Cₖ|) Σxᵢ 구현

코드완성형 [응용] ⭐⭐ Slide 5
아래는 맨해튼 거리(L1)를 계산하는 코드이다. 빈칸 ①②를 채우고, 유클리드 거리 코드와 비교하여 달라진 부분이 무엇인지 설명하시오.
def manhattan_distance(x, z): total = 0 for i in range(len(x)): total += ① return ② # 검증: x=(4,3), z=(2,3) → 결과: 2 print(manhattan_distance([4,3], [2,3]))
정답
① abs(x[i] - z[i]) — 절댓값 차이 (제곱이 아님)
② total — 제곱근 없이 합계 그대로 반환

유클리드와의 차이:
• 유클리드: ① (x[i]-z[i])**2 → ② math.sqrt(total)
• 맨해튼: ① abs(x[i]-z[i]) → ② total
핵심 차이는 제곱/제곱근 연산의 유무이다. 맨해튼은 절댓값 합이므로 이상치 하나가 거리를 폭발적으로 늘리지 않아 더 강건하다.

유클리드 거리 코드(Q08)에서 거리 측도를 L1으로 변경. 수식 구조는 같으나 연산자가 달라지며, 차이점 설명을 추가로 요구하는 심화 문제.

코드완성형 [응용] ⭐⭐⭐ Slide 6
아래는 KNN의 회귀(Regression) 버전 예측 함수이다. 분류와 달리 이웃의 출력값 평균을 반환한다. 빈칸 ①②를 채우시오.
def knn_regression(X_train, y_train, x_test, k): distances = [(euclidean_distance(x, x_test), y_train[i]) for i, x in enumerate(X_train)] distances.sort(key=lambda x: x[0]) k_neighbors = distances[:k] # 분류와 다른 부분: 다수결 대신 평균 neighbor_values = [ ① ] return ② # 검증: 이웃 3개의 y값이 [2.0, 4.0, 3.0]이면 → 3.0 반환
정답
① val for _, val in k_neighbors
— 거리/레이블 튜플에서 레이블(연속값) 추출

② sum(neighbor_values) / len(neighbor_values)
— 이웃 k개 출력값의 산술 평균 → ŷ = (1/K) Σⱼ yᵢⱼ 구현

분류 vs 회귀 비교:
분류: Counter로 다수결 → 가장 많은 클래스 반환
회귀: 이웃 y값들의 평균 → 연속값 반환

코드완성형 [응용] ⭐⭐⭐ Slide 8, 10
아래는 k-fold 교차검증으로 최적 K를 탐색하는 코드이다. 빈칸 ①~③을 채우시오.
(KNN의 K 선택 + 교차검증 개념을 결합한 문제)
def find_best_k(X, y, k_range, n_folds=5): best_k, best_acc = 1, 0.0 fold_size = len(X) // n_folds for k in k_range: fold_accs = [] for f in range(n_folds): # 검증 폴드 분리 val_X = X[f*fold_size : ① ] val_y = y[f*fold_size : (f+1)*fold_size] train_X = X[:f*fold_size] + X[ ① :] train_y = y[:f*fold_size] + y[(f+1)*fold_size:] # 각 검증 샘플 예측 & 정확도 계산 correct = sum( knn_predict(train_X, train_y, x, k) == true_y for x, true_y in zip(val_X, val_y) ) fold_accs.append( ② ) avg_acc = sum(fold_accs) / n_folds if avg_acc > best_acc: best_acc = avg_acc best_k = k # ③ 홀수 K 보정 if best_k % 2 == ③ : best_k += 1 return best_k
정답
① (f+1)*fold_size — 현재 폴드의 끝 인덱스
② correct / len(val_X) — 해당 폴드의 정확도 = 맞은 수 / 전체 검증 수
③ 0 — 짝수인 경우(% 2 == 0) +1로 홀수 보정 (동점 방지)

교차검증 흐름: 전체 데이터를 n_folds개로 분할 → 각 폴드를 검증 데이터로 순서대로 사용 → 평균 정확도가 최고인 K 선택 → 홀수 보정.
변형 포인트
Slide 8(K 선택: 교차검증)과 Slide 10(실용적 체크리스트: 홀수 K 권장)을 결합한 개념 결합 문제. 단순 거리 계산을 넘어 전체 선택 파이프라인을 코드로 이해해야 함.

서술형 ⭐⭐ Slide 9
'차원의 저주(Curse of Dimensionality)'가 KNN 성능에 미치는 영향을 세 가지 문제 유형으로 나누어 설명하고, 그 해결 방향을 서술하시오.
핵심 답안
① 거리 집중 현상: 차원이 높아질수록 모든 점들이 거의 같은 거리에 위치하게 된다. '가장 가까운 이웃'과 '가장 먼 이웃'의 거리 차이가 무의미해져 이웃 선택이 불가능해진다.

② 노이즈 특성 증가: 변수가 늘어날수록 예측과 관련 없는 특성(노이즈)이 포함될 확률이 높아진다. 예를 들어 혈액형, 병원 방문 요일 같은 무관한 특성이 거리 계산을 왜곡하여 관련 없는 이웃을 선택하게 한다.

③ 계산 비용 증가: 거리 계산 비용이 O(n·d)이므로 차원 d가 커질수록 예측 속도가 크게 저하된다.

해결 방향: 특성 선택(Feature Selection)으로 무관한 특성을 제거하거나, PCA(주성분 분석)로 차원을 축소한 뒤 KNN을 적용하면 성능이 크게 향상된다.

서술형 ⭐⭐ Slide 14
K-means 알고리즘이 반드시 수렴함을 직관적으로 증명하시오. 또한 수렴이 보장됨에도 불구하고 실전에서 여러 번 실행하는 이유를 설명하시오.
핵심 답안
[수렴 증명]
1. 할당 단계: 각 xᵢ를 가장 가까운 중심에 배정하므로 J는 감소하거나 같아진다 (J_new ≤ J_old).
2. 갱신 단계: 클러스터 중심을 평균으로 갱신하면 평균은 제곱 오차를 최소화하는 점이므로 J가 감소하거나 같아진다.
3. 유한성: 클러스터 할당의 경우의 수는 Kⁿ 개로 유한하다. J가 단조 감소하고 경우의 수가 유한하므로 반드시 수렴한다.

[여러 번 실행하는 이유]
수렴은 보장되지만 전역 최솟값(global minimum)은 보장되지 않는다. 초기 중심 선택에 따라 서로 다른 지역 최솟값(local minimum)에 빠질 수 있다. 따라서 다수의 랜덤 초기화를 시도하고 그 중 J가 가장 작은 결과를 최종 답으로 채택한다(n_init 파라미터).

서술형 ⭐⭐ Slide 3, 18
KNN과 K-means를 학습 유형, K의 의미, 훈련 단계, 예측 방식 네 가지 관점에서 비교 서술하시오.
핵심 답안
① 학습 유형: KNN은 레이블이 있는 데이터로 P(y|x)를 추정하는 지도학습이고, K-means는 레이블 없이 P(x)의 구조를 발견하는 비지도학습이다.

② K의 의미: KNN에서 K는 탐색할 이웃의 개수(탐색 범위)를 의미하고, K-means에서 K는 데이터를 나눌 클러스터의 개수(그룹 수)를 의미한다.

③ 훈련 단계: KNN은 별도의 파라미터 학습이 없는 Lazy Learning으로 훈련 비용이 O(1)이다. K-means는 할당-갱신을 반복하는 최적화 과정이 훈련 단계에 해당한다.

④ 예측 방식: KNN은 K개 이웃의 다수결(분류) 또는 평균(회귀)으로 예측한다. K-means는 새 데이터를 가장 가까운 centroid에 배정하여 클러스터 레이블을 부여한다.

서술형 [응용] ⭐⭐⭐ Slide 17
K-means의 한계 중 "구형(원형) 클러스터 가정"과 "이상치 민감성"이 실제 문제에서 어떻게 나타나는지 구체적 예시를 들어 설명하고, 각각의 대안 알고리즘과 그 이유를 서술하시오.
핵심 답안
[구형 클러스터 가정]
K-means는 유클리드 거리를 최소화하므로 클러스터가 원형(구형)으로 형성된다고 가정한다. 예를 들어 초승달 모양이나 고리 모양으로 분포된 데이터(두 개의 동심원)의 경우, K-means는 올바른 클러스터를 찾지 못하고 단순히 좌/우로 나누는 잘못된 결과를 낸다.
→ 대안: DBSCAN — 밀도 기반 클러스터링으로 임의 모양의 클러스터를 포착 가능. 또는 GMM(가우시안 혼합 모델)로 타원형 클러스터 처리 가능.

[이상치 민감성]
K-means의 centroid는 클러스터 데이터의 '평균'이므로, 이상치 하나가 중심을 크게 왜곡한다. 예를 들어 {1,2,3,100}이라는 데이터가 있으면 100 하나 때문에 centroid가 26.5로 실제 군집(1~3) 중심에서 크게 벗어난다.
→ 대안: K-medoids — centroid 대신 실제 데이터 포인트 중 중앙값(medoid)을 중심으로 사용하므로 이상치 영향을 최소화한다.
변형 포인트
Slide 17의 한계 목록을 단순 열거하는 것을 넘어, 각 한계가 실제 데이터 상황에서 어떻게 발현되는지 예시를 들고 대안의 이유까지 설명해야 하는 심화 문제.

서술형 [응용] ⭐⭐⭐ Slide 9, 16, 19
실제 ML 프로젝트에서 고차원 고객 데이터(100개 특성, 10만 건)로 고객 세분화를 수행하려 한다.
KNN과 K-means 중 어느 알고리즘을 어느 단계에서 사용해야 하는가?
차원의 저주, 레이블 유무, 목적(세분화 vs 분류)을 모두 고려하여 파이프라인을 제시하고 각 선택의 이유를 서술하시오.
핵심 답안
[권장 파이프라인]
1단계 전처리: Z-score 표준화 (거리 기반 알고리즘 모두 스케일 민감)
2단계 차원 축소: PCA로 100차원 → 10~20차원 축소 (차원의 저주 해소)
3단계 군집화: K-means 적용 → Elbow Method로 최적 K 결정
4단계 (선택): 군집 레이블이 생성된 이후, 신규 고객 분류에 KNN 적용

[선택 이유]
• 고객 세분화 목적이므로 레이블이 없는 상태에서 구조를 찾아야 한다 → K-means (비지도)
• 100차원 원시 데이터에 KNN을 바로 적용하면 차원의 저주로 거리 의미가 없어지고 예측 비용 O(n·d)도 폭발 → PCA 선행 필수
• K-means로 세분화한 결과(클러스터 레이블)가 생기면, 이후 신규 고객 분류에는 KNN을 지도학습으로 활용 가능
• 10만 건 × 100차원에 KNN 단독 사용 시 예측 1건당 O(100,000 × 100) = 10M 연산 → 비실용적

서술형 [응용] ⭐⭐⭐ Slide 8, 14
아래는 어떤 모델 A의 훈련/테스트 성능 결과이다. 이 패턴이 나타난 원인을 KNN의 어떤 설정에서 비롯된 것인지 진단하고, 해결 방안을 구체적으로 서술하시오.

- 훈련 정확도: 100%
- 테스트 정확도: 52% (거의 랜덤 수준)
- 사용 알고리즘: KNN
- 훈련 데이터 크기: n=500
핵심 답안
[원인 진단 — K=1 과적합]
훈련 정확도 100%는 KNN에서 K=1일 때 발생하는 전형적인 증상이다. K=1이면 각 훈련 데이터가 자기 자신의 유일한 이웃이 되므로 훈련 오류가 정확히 0이 된다. 그러나 테스트 데이터에 대해서는 새로운 패턴에 과적합된 매우 복잡한 결정 경계가 적용되어 52%라는 랜덤에 가까운 정확도를 보인다.

이는 Slide 8의 K=1: 낮은 편향/높은 분산 상황에 해당하며, 과적합(Overfitting)의 교과서적 사례다.

[해결 방안]
1. K 값 증가: √500 ≈ 22를 시작점으로 하여 홀수 K를 탐색
2. k-fold 교차검증: 5-fold 또는 10-fold 교차검증으로 훈련 데이터 내에서 검증하며 최적 K 선택
3. 정규화 확인: 스케일 미정규화 시 거리 계산 왜곡 → Min-Max 또는 Z-score 적용
4. 차원 점검: 특성 수가 많으면 PCA 선처리 후 KNN 적용
변형 포인트
Slide 8(K=1 과적합)의 개념을 역방향으로 적용. 수치(훈련 100%, 테스트 52%)를 보고 K=1임을 역으로 진단하고 해결책을 제시하는 문제. 훈련=100%라는 단서가 핵심.
