Data

Data 어원 & 정의
  • datum의 복수형
    • transmissible & storable computer information
    • data processing이라는 표현은 1954년에 처음 사용
    • big data 문맥에서는 information과 동의어
  • 데이터는 이산적(discrete) 값들로 구성
    • 수량·품질·사실·통계 등 정보를 전달
DIKW 피라미드

데이터 → 정보 → 지식 → 지혜의 위계

Dataset 구성 요소

Instance (→ 행)

  • 데이터셋의 각 행(row) 을 나타냄
  • 단일 개체를 설명하는 여러 attribute 포함
  • aka: object, record, point, vector, pattern, case, event, sample, observation, entity

Attribute (↓ 열)

  • 데이터셋의 각 열(column) 을 나타냄
  • instance의 특정 속성(property)을 설명
  • aka: variable, characteristic, field, feature, dimension

03

Exploratory Data Analysis

EDA란?
  • 데이터 과학 분야의 기초적 접근법
  • 데이터를 분석·요약·이해하는 데 필수
  • 데이터의 숨겨진 구조, 패턴, 이상치, 인사이트를 발굴
  • 공식적인 모델링 이전에 적용되는 탐색적 과정
  • John Wilder Tukey (1915–2000) 가 개념 정립
① Revelation
탐색 단계에서 데이터가 스스로 이야기를 드러내야 함
e.g., histogram, scatter plot, boxplot으로 트렌드·이상치·비정상 분포 시각화
② Resistance
분석은 이상치·오류·비정상 데이터에 강건(robust)해야 함
e.g., 평균·표준편차 대신 median + IQR 사용
③ Re-expression
패턴을 더 명확히 드러내도록 데이터를 변환·재구성
e.g., 왜곡된 데이터에 log 변환 적용 → 정규화
④ Residuals
관측값과 예측값의 차이(잔차)를 분석
e.g., 모델이 데이터에 얼마나 잘 맞는지 평가에 활용

04

Data Preprocessing

전처리란?
  • 데이터 마이닝·머신러닝에서 필수적인 단계
  • 원시 데이터를 깨끗하고 정돈된 형식으로 변환
  • 데이터 품질 향상 → 특정 작업에 적합한 형태로 가공

실세계 데이터의 특성

  • Incomplete (결측) — 속성·값이 누락되거나 집계 데이터만 존재
  • Noisy (노이즈) — 오류나 이상치 포함
  • Inconsistent (불일치) — 코드·이름 차이 등 불일치
핵심 작업 (Key Tasks)
  • Data Cleansing
    • 결측값 처리, 노이즈·이상치 제거, 불일치 수정
  • Data Transformation
    • 정규화(Normalization), 표준화(Standardization), 범주형 변수 인코딩
  • Data Reduction
    • 핵심 정보를 보존하면서 데이터 볼륨 축소
  • Data Integration
    • 여러 소스의 데이터를 통합된 데이터셋으로 결합

05

Summary

강의 핵심 정리
📊

Data & Dataset

데이터는 이산적 값들로 구성된 정보. 데이터셋은 Instance(행)와 Attribute(열)로 구성되며, Numerical/Categorical로 분류됨

🏷️

Data Creation

CVAT, LabelMe 등 어노테이션 도구로 레이블 데이터 생성. AI-Hub, 공공데이터포털 등 공개 저장소 활용 가능

🔍

EDA

Tukey가 정립한 탐색적 데이터 분석. Revelation·Resistance·Re-expression·Residuals의 4원칙으로 데이터 구조 탐색

🧹

전처리 필요성

실세계 데이터는 불완전·노이즈·불일치. 전처리 없이는 Garbage In, Garbage Out!

⚙️

전처리 4단계

Cleansing → Integration → Transformation → Reduction 순서로 원시 데이터를 분석 가능한 형태로 가공

⏱️

전처리의 비중

데이터 과학자 업무 시간의 약 60%가 데이터 정제·정리에 소요됨. 모델 성능·일반화 능력에 직결

전처리 유형 비교
유형 목적 주요 기법 해결하는 문제
Cleansing 데이터 품질 향상 결측값 대체(imputation), 이상치 제거, 중복 제거 Incomplete, Noisy
Integration 데이터 통합 병합(Merge), 조인(Join), 스키마 매핑 Inconsistent (다중 소스)
Transformation 형태 변환 정규화(Min-Max), 표준화(Z-score), One-hot Encoding, Log 스케일 불일치, 범주형 변수
Reduction 차원 축소 PCA, 특징 선택(Feature Selection), 샘플링 고차원, 과적합 위험
