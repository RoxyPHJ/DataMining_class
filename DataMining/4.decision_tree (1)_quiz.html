Q01 객관식 ⭐ Slide 2–3/16

다음 중 분류(Classification)에 대한 설명으로 올바른 것은?

A 연속적인 수치 값을 출력한다.
B 범주형(이산형) 클래스 레이블을 예측한다.
C 비지도학습의 한 형태이다.
D 레이블 없는 데이터로부터 패턴을 발견한다.
📖 정답 및 해설 보기▼
정답: B
분류는 범주형(categorical) 클래스 레이블을 예측하는 데이터 분석 태스크입니다. 예: yes/no, spam/not spam, A/B/C. 연속 값 예측은 수치 예측(회귀)이며, 분류는 레이블된 데이터를 사용하는 지도학습입니다.
Q02 객관식 ⭐⭐ Slide 6/16

Decision Tree의 각 구성 요소에 대한 설명으로 올바르지 않은 것은?

A Internal Node는 속성에 대한 테스트를 나타낸다.
B Branch는 테스트 결과에 해당하는 경로이다.
C Leaf Node는 속성 테스트 조건을 포함한다.
D Root Node는 트리의 최상단 노드이다.
📖 정답 및 해설 보기▼
정답: C
Leaf Node는 속성 테스트가 아니라 클래스 레이블(class label)을 포함합니다. 새로운 인스턴스를 분류할 때 Root Node에서 시작하여 각 Internal Node의 테스트 조건을 평가하고, Branch를 따라 이동하여 최종적으로 Leaf Node에 도달하면 그 레이블이 예측 클래스가 됩니다.
Q03 객관식 ⭐⭐ Slide 8/16

다음 중 CART 알고리즘의 특징으로 올바른 것은?

A Information Gain을 분할 기준으로 사용한다.
B 이진 트리(Binary Tree)만 생성하며 분류와 회귀를 모두 지원한다.
C 범주형 속성만 처리할 수 있다.
D 가지치기(pruning)를 수행하지 않는다.
📖 정답 및 해설 보기▼
정답: B
CART(Classification And Regression Trees)는 이진 트리(Binary Tree)만 생성하며, 분류에는 Gini Index, 회귀에는 MSE/RSS를 사용합니다. ID3는 Information Gain만 사용하고 범주형 속성만 처리합니다. C4.5는 범주형+연속형 모두 지원합니다.
Q04 객관식 ⭐⭐⭐ Slide 13/16

14개 튜플로 구성된 데이터셋 D에서 yes=9, no=5일 때, Info(D)의 값으로 올바른 것은?

A 0.940 bits
B 0.694 bits
C 0.246 bits
D 0.459 bits
📖 정답 및 해설 보기▼
정답: A — 0.940 bits
엔트로피 공식을 적용합니다:
Info(D) = −(9/14)·log₂(9/14) − (5/14)·log₂(5/14)
= −0.6429·(−0.6375) − 0.3571·(−1.485)
= 0.4098 + 0.5302 = 0.940 bits
0.694는 age로 분할 후 조건부 엔트로피, 0.246은 Gain(age), 0.459는 Gini(D)입니다.
Q05 객관식 ⭐⭐⭐ Slide 15/16

Gain Ratio가 Information Gain보다 유리한 이유로 가장 올바른 것은?

A 계산이 더 단순하여 속도가 빠르다.
B 연속형 속성을 처리할 수 있다.
C 이진 트리만 생성하여 단순하다.
D 값이 많은 속성에 대한 편향을 줄여 더 나은 일반화를 제공한다.
📖 정답 및 해설 보기▼
정답: D
Information Gain은 값이 많은 속성(예: product_ID)을 과도하게 선호합니다. Gain Ratio는 Split Information으로 정규화하여 이 편향을 줄입니다. Split Information은 데이터가 얼마나 고르게 분할되는지를 측정하며, 값이 너무 많으면 SplitInfo가 커져 GainRatio가 작아집니다.

⭕ 참(O)
❌ 거짓(X)
📖 정답 및 해설 보기▼
정답: O (참)
지도학습의 핵심 특성입니다. 훈련 데이터 D = {(x₁,y₁), (x₂,y₂), …, (xₙ,yₙ)}로 f: x → y 매핑을 학습하고, 학습된 모델을 훈련 세트에 없는 새로운 입력에 적용하여 예측합니다.
Q07 OX 문제 ⭐ Slide 6/16

Decision Tree는 회귀(Regression) 문제에는 사용할 수 없고, 분류(Classification) 문제에만 사용 가능하다.

⭕ 참(O)
❌ 거짓(X)
📖 정답 및 해설 보기▼
정답: X (거짓)
Decision Tree는 분류와 회귀 모두에 사용 가능한 지도학습 방법입니다. 특히 CART(Classification And Regression Trees) 알고리즘은 이름에서도 알 수 있듯이 분류에는 Gini Index를, 회귀에는 MSE/RSS를 분할 기준으로 사용합니다.
Q08 OX 문제 ⭐⭐ Slide 9/16

Generate_decision_tree 알고리즘에서 파티션 D_j가 비어있을 경우(empty), 해당 노드를 D 전체의 다수결 클래스로 레이블된 Leaf Node로 처리한다.

⭕ 참(O)
❌ 거짓(X)
📖 정답 및 해설 보기▼
정답: O (참)
알고리즘 12–13번 줄에 해당합니다: if D_j is empty then attach a leaf labeled with the majority class in D. 분할 결과 해당 브랜치에 데이터가 없을 때, 부모 노드 D 전체의 다수결 클래스를 사용하는 majority voting 방식입니다.
Q09 OX 문제 ⭐⭐ Slide 11/16

Gini Impurity가 0일 때, 데이터셋은 완전히 혼합된(mixed) 상태를 의미한다.

⭕ 참(O)
❌ 거짓(X)
📖 정답 및 해설 보기▼
정답: X (거짓)
Gini Impurity가 0일 때는 데이터셋이 완전히 순수(pure)한 상태, 즉 모든 튜플이 동일한 클래스에 속하는 상태를 의미합니다. Gini 값이 높을수록 클래스가 더 혼합된(mixed) 상태입니다.
Q10 OX 문제 ⭐⭐⭐ Slide 13/16

14-tuple 고객 데이터셋에서 age 속성으로 분할 시, middle_aged 파티션(4 yes, 0 no)은 엔트로피가 0이므로 즉시 Leaf Node가 된다.

⭕ 참(O)
❌ 거짓(X)
📖 정답 및 해설 보기▼
정답: O (참)
Info(middle_aged) = −(4/4)·log₂(4/4) = 0. 모든 튜플이 동일한 클래스(yes)이므로 완전히 순수(pure)한 파티션입니다. 알고리즘 2–3번 줄: "if tuples are all of the same class C, then return N as a leaf node labeled C"에 의해 즉시 Leaf Node가 됩니다.
Q11 단답형 ⭐ Slide 8/16

C4.5 알고리즘이 Information Gain의 편향 문제를 해결하기 위해 사용하는 속성 선택 측도는 무엇인가?

📖 정답 및 해설 보기▼
정답: Gain Ratio (이득 비율)
Gain Ratio는 Information Gain을 Split Information으로 나눈 값입니다:
GainRatio(A) = Gain(A) / SplitInfo_A(D)
Split Information은 속성이 데이터를 얼마나 고르게 분할하는지 측정하여, 값이 많은 속성에 대한 편향을 완화합니다.
Q12 단답형 ⭐⭐ Slide 13/16

14-tuple 데이터셋에서 Info(D) = 0.940 bits, Info_age(D) = 0.694 bits일 때, Gain(age)의 값은? (단위 bits 생략 가능)

📖 정답 및 해설 보기▼
정답: 0.246 bits
Gain(age) = Info(D) − Info_age(D) = 0.940 − 0.694 = 0.246 bits
4개 속성 중 Gain(age) = 0.246이 가장 높으므로 age가 루트 노드 분할 속성으로 선택됩니다.
Q13 단답형 ⭐ Slide 8/16

CART 알고리즘이 분류 문제에서 사용하는 속성 선택 측도를 영문으로 쓰시오.

📖 정답 및 해설 보기▼
정답: Gini Index (Gini Impurity)
CART는 분류에 Gini Index, 회귀에 MSE/RSS를 사용합니다:
Gini(D) = 1 − Σᵢ pᵢ²
Q14 단답형 ⭐⭐⭐ Slide 16/16

14-tuple 데이터셋에서 yes=9, no=5일 때, Gini(D)의 값을 소수점 3자리로 쓰시오.

📖 정답 및 해설 보기▼
정답: 0.459
Gini(D) = 1 − (9/14)² − (5/14)²
= 1 − (0.6429)² − (0.3571)²
= 1 − 0.4133 − 0.1276
= 1 − 0.5408 = 0.459
Q15 단답형 ⭐ Slide 8/16

Information Gain을 분할 기준으로 사용하며, 범주형 속성만 처리하고 가지치기를 수행하지 않는 Decision Tree 알고리즘의 이름은?

📖 정답 및 해설 보기▼
정답: ID3
ID3(Iterative Dichotomizer)는 Quinlan(1986)이 제안한 알고리즘입니다. Information Gain을 분할 기준으로 사용하며, 범주형 속성만 처리하고, 사후 가지치기(post-pruning)를 수행하지 않습니다. C4.5는 ID3의 확장 버전입니다.
Q16 코드 완성 ⭐⭐ Slide 9/16

아래 Generate_decision_tree 알고리즘의 빈칸을 채우시오. 이산 속성에서 다지 분할을 허용하는 조건을 나타냅니다.

if splitting_attribute is discrete-valued and
     allowed then // not restricted to binary trees
    attribute_list ← attribute_list − splitting_attribute;
📖 정답 및 해설 보기▼
정답: multiway
multiway splits allowed — 이 조건이 참이면 이산 속성의 각 값에 대해 별도의 브랜치를 생성하는 다지 분할(multiway split)을 허용합니다. 이 조건이 false이면 이진 분할(binary split)만 수행합니다.
Q17 코드 완성 ⭐⭐ Slide 11/16

아래 수식의 빈칸에 알맞은 개념 이름을 쓰시오. 이 값이 클수록 데이터셋이 불순함을 나타냅니다.

// Attribute Selection Measure: of D

Info(D) = −Σᵢ pᵢ · log₂(pᵢ), where pᵢ = |Cᵢ,D| / |D|
📖 정답 및 해설 보기▼
정답: Entropy (엔트로피)
Entropy(엔트로피)는 데이터셋 D의 불순도(impurity)를 측정합니다. 모든 튜플이 같은 클래스이면 entropy = 0(완전 순수), 클래스가 균등하게 분포되면 entropy가 최대가 됩니다. ID3 알고리즘에서 Information Gain을 계산할 때 기반이 됩니다.
Q18 코드 완성 ⭐⭐⭐ Slide 9/16

아래 알고리즘에서 모든 튜플이 동일한 클래스 C일 때의 처리를 나타내는 코드입니다. 빈칸을 채우시오.

if tuples in D are all of the same class, C, then
    return N as a labeled with class C;
📖 정답 및 해설 보기▼
정답: leaf (Leaf Node)
파티션의 모든 튜플이 동일한 클래스 C에 속할 때, 더 이상 분할이 필요 없으므로 해당 노드를 클래스 C로 레이블된 Leaf Node로 반환합니다. 이것이 Decision Tree 귀납 알고리즘의 기저 조건(base case) 중 하나입니다.
Q19 서술형 ⭐⭐⭐ Slide 11/16

Information Gain, Gain Ratio, Gini Index 세 가지 속성 선택 측도를 공식, 사용 알고리즘, 특징/장단점을 중심으로 비교 서술하시오.

📖 모범 답안 보기▼
모범 답안
① Information Gain (ID3)
공식: Gain(A) = Info(D) − Info_A(D)
Info(D) = −Σᵢ pᵢ·log₂(pᵢ)로 전체 엔트로피, Info_A(D) = Σⱼ (|Dⱼ|/|D|)·Info(Dⱼ)로 속성 A로 분할 후 조건부 엔트로피를 계산. 분할 후 불확실성이 가장 많이 감소하는 속성 선택.
단점: 값이 많은 속성(예: product_ID)에 편향됨.

② Gain Ratio (C4.5)
공식: GainRatio(A) = Gain(A) / SplitInfo_A(D)
SplitInfo_A(D) = −Σⱼ (|Dⱼ|/|D|)·log₂(|Dⱼ|/|D|)로 분할 방식 자체의 정보량 측정.
장점: 정규화를 통해 값이 많은 속성에 대한 편향 감소. ID3 개선.

③ Gini Index (CART)
공식: Gini(D) = 1 − Σᵢ pᵢ²
무작위 레이블 선택 시 오분류 확률로 해석. 이진 분할(binary split)에 최적화.
특징: 이진 트리 생성, 분류+회귀 모두 지원.
Q20 서술형 ⭐⭐⭐ Slide 6–7/16

Decision Tree의 장점(Advantages) 4가지와 한계(Limitations) 2가지를 각각 서술하고, 한계를 극복하기 위한 방법을 간략히 언급하시오.

📖 모범 답안 보기▼
모범 답안
장점(Advantages):
① 도메인 지식이나 파라미터 튜닝 불필요
② 다차원 데이터(multidimensional data) 처리 가능
③ 인간이 직관적으로 이해하기 쉬운 if-then 규칙 형태
④ 계산 효율적이며 실제로 좋은 성능 달성 가능

한계(Limitations):
① 과적합(Overfitting): 훈련 데이터에 너무 잘 맞는 복잡한 트리 생성 가능
  → 극복 방법: 가지치기(Pruning) — 사전 가지치기(Pre-pruning) 또는 사후 가지치기(Post-pruning, C4.5)
② 노이즈/이상값 반영: 일부 브랜치가 노이즈나 이상값에 의해 형성될 수 있음
  → 극복 방법: 앙상블 기법(Random Forest, Boosting) 사용
