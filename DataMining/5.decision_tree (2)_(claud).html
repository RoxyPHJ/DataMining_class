01
Decision Tree Pruning — 가지치기 개요
⚠ 왜 가지치기가 필요한가? 결정 트리를 완전히 성장시키면 훈련 데이터에 과적합(overfitting)되어 노이즈·이상치를 암기하게 되고, 새로운 데이터에 대한 예측 성능이 떨어진다.
🚨 완전 성장 트리의 문제점
  • Overfitting — 노이즈·이상치에 과적합
  • 과도한 복잡성 — 트리가 너무 복잡해짐
  • 낮은 일반화 — 미학습 데이터에서 성능 저하

핵심 직관: 매우 세밀한 트리는 훈련 데이터를 암기(memorize)하지만 일반화(generalize)에 실패한다.

✂ 가지치기(Pruning)의 효과
  • 트리를 더 작게 만든다
  • 해석하기 쉬워진다
  • 예측 속도가 빨라진다
  • 테스트 데이터 정확도 향상
핵심 모델 복잡성과 일반화 능력 사이의 균형을 맞추는 것이 가지치기의 목적이다.
🌳 가지치기 전후 비교 (SVG 시각화)

BEFORE — 완전 성장 트리 (과적합)

AFTER — 가지치기 후 트리 (일반화)

02
Pruning 두 가지 접근법
🔵 Pre-Pruning (사전 가지치기)
Early Stopping
  • 트리 구성 도중에 분할을 중단
  • 기준 사용 (Information Gain, Gini)
  • 분할이 유의미하지 않으면 노드를 리프로 전환
  • 위험: 너무 일찍 중단 → 과소적합(Underfitting)
🟣 Post-Pruning (사후 가지치기)
Backward Pruning
  • 먼저 완전한 트리를 생성
  • 이후 불필요한 서브트리를 제거
  • 서브트리를 리프(최빈 클래스)로 교체
  • 더 신뢰할 수 있으나 계산 비용이 높음
구분Pre-PruningPost-Pruning
시점트리 생성 중트리 완성 후
방법조기 분할 중단서브트리 제거
속도빠름느림 (완전 트리 필요)
위험과소적합(Underfitting)높은 계산 비용
신뢰성보통더 신뢰할 수 있음
핵심 아이디어 Pre-Pruning은 빠르지만 보수적일 수 있고, Post-Pruning은 신뢰할 수 있지만 비용이 높다. 두 방법 모두 모델 복잡성과 일반화 능력의 균형을 목표로 한다.
03
Pruning 대표 방법론
① Cost Complexity Pruning — CART

오류율과 트리 복잡성을 결합한 비용 함수를 최소화한다.

Rα(T) = R(T) + α · |T|
  • R(T): 트리의 훈련 오류 (또는 불순도)
  • |T|: 리프 노드의 수 (모델 복잡성)
  • α: 복잡도 패널티 파라미터
  • 서브트리 vs 가지치기 버전을 비교하여 더 낮은 비용을 가진 것 선택
  • 최적 α는 별도의 가지치기 데이터셋으로 선택
② Pessimistic Pruning — C4.5
  • 훈련 데이터만 사용 (별도 가지치기 세트 불필요)
  • 보다 현실적인 오류 추정을 위해 패널티를 추가
  • 수정된 리프 오류가 서브트리 오류보다 낮으면 가지치기 수행
③ MDL-based Pruning
Minimum Description Length
  • 총 인코딩 길이를 최소화하는 모델 선택
  • 총 인코딩 길이 ≈ 모델 복잡성 + 정밀도 오류
  • 서브트리를 리프로 대체했을 때 전체 설명 길이가 감소하면 가지치기 수행
  • 트리 크기와 데이터 피팅을 모두 고려하여 균형을 맞춤
방법론데이터 필요핵심 기준특징
Cost Complexity (CART)별도 가지치기 세트R(T) + α·|T|α로 복잡도 조절
Pessimistic (C4.5)훈련 데이터만수정된 오류율빠르고 간편
MDL훈련 데이터만최소 설명 길이정보 이론 기반
04
모델 평가 및 선택
핵심 질문 분류기를 만든 후 반드시 물어야 할 것: (1) 모델이 얼마나 정확한가? (2) 미학습 데이터에 일반화할 수 있는가? (3) 여러 모델을 어떻게 비교하는가?
📊 훈련 정확도의 한계
  • 훈련 정확도는 과적합으로 인해 종종 지나치게 낙관적이다
  • 따라서 평가는 반드시 미학습(Unseen) 테스트 데이터에서 수행되어야 한다

모델 평가 프로세스

1. 데이터 분할
훈련 세트와 테스트 세트로 분리

2. 모델 평가
고훈련 정확도 + 과적합 감지

3. 모델 비교
다수의 모델을 정확도·안정성·일반화로 비교

4. 최적 모델 선택
정확도, 안정성, 일반화 등 종합 고려

혼동 행렬 (Confusion Matrix)

TPTrue Positive

Actual −

  • TP: 양성을 양성으로 올바르게 예측
  • TN: 음성을 음성으로 올바르게 예측
  • FP: 오경보 (음성 → 양성, False Alarm)
  • FN: 놓친 양성 (양성 → 음성, Miss)
📝 참고 (Note)
Accuracy클래스가 균형 잡혀 있을 때 잘 작동한다.
Sensitivity (Recall)양성이 얼마나 잘 감지되는지 측정한다.
Specificity음성이 얼마나 잘 감지되는지 측정한다.
Measure공식설명
Accuracy (정확도)(TP+TN) / (P+N)전체 중 올바르게 예측한 비율
Error Rate (오류율)(FP+FN) / (P+N)전체 중 잘못 예측한 비율
Sensitivity / Recall (재현율)TP / P실제 양성 중 올바르게 탐지된 비율
Specificity (특이도)TN / N실제 음성 중 올바르게 탐지된 비율
Precision (정밀도)TP / (TP+FP)양성 예측 중 실제 양성 비율
F₁-score2×precision×recall / (precision+recall)정밀도와 재현율의 조화 평균
Fβ-score(1+β²)×precision×recall / (β²×precision+recall)β로 precision/recall 가중치 조절
06
클래스 불균형 문제 (Class Imbalance)
⚠ 정확도의 함정 데이터의 97%가 음성이라면, 모든 샘플을 음성으로 예측하는 모델도 97% 정확도를 달성한다. 하지만 이 모델은 실제 양성 케이스를 전혀 감지하지 못한다!
실제 불균형 데이터 예시
  • 사기 탐지 — 정상 거래 99%+, 사기 1% 미만
  • 암 진단 — 정상 환자 다수, 암 환자 소수
  • 침입 탐지 — 정상 트래픽 다수, 공격 소수
문제점 모델이 소수(Minority) 클래스를 무시하는데, 의료 진단이나 보안 등 중요한 응용에서 양성 케이스 누락(False Negative)은 매우 심각하다.
✅ 해결책 — 클래스에 집중하는 지표 사용
  • Precision (정밀도): 양성 예측의 신뢰성
  • Recall (재현율/Sensitivity): 양성(희귀 클래스) 감지 능력
  • F₁-score: 정밀도와 재현율의 균형
07
신뢰할 수 있는 평가 방법
목표 미학습 데이터에서의 일반화 성능을 신뢰할 수 있게 추정하고, 평가의 편향과 분산을 줄이며, 서로 다른 모델 간 공정한 비교를 보장하는 것.
① Holdout Method
  • 데이터를 훈련 세트와 테스트 세트로 분할
  • 간단하고 빠름
  • 분할 방식에 따라 결과가 달라질 수 있음
② Cross-Validation
  • 데이터를 여러 Fold로 분할
  • 서로 다른 분할에서 반복 훈련·테스트
  • 더 안정적이고 신뢰할 수 있는 추정
③ Bootstrap
  • 복원 추출(w/ replacement)로 샘플링
  • 여러 재샘플링 데이터셋에서 성능 평가
  • 데이터가 제한적일 때 유용

5-Fold Cross-Validation

데이터를 5개의 Fold로 나누고, 각 Fold를 번갈아 테스트 세트로 사용한다.

5번의 훈련-테스트를 반복하여 성능을 평균낸다.

장점: 모든 데이터가 훈련과 테스트에 모두 사용된다.

k-Fold CV일반적으로 k=10을 사용하며, 편향과 분산 간의 좋은 균형을 제공한다.
08
앙상블 방법 소개
앙상블 학습이란? 여러 모델 M₁, M₂, …, Mₖ를 결합하여 단일 모델보다 강력한 복합 모델을 만들고 분류 성능을 향상시키는 방법이다. 각 분류기는 주어진 데이터의 클래스 레이블에 "투표"한다.
📋 기본 프로세스
  • 여러 훈련 세트 D₁, D₂, …, Dₖ 생성
  • 기본 분류기 Mᵢ 훈련
  • 각 모델이 예측값을 생성
  • → 최종 예측은 결합된 출력에 기반
🎯 앙상블 학습의 목표
  • 단일 모델보다 더 강력한 복합 모델 구축
  • 분류 성능 향상
  • 다양성(Diversity)을 통해 정확도 개선
  • 다수 모델이 틀릴 때만 앙상블이 오류 발생

앙상블 구조

원본 데이터 D에서 여러 훈련 세트 D₁~Dₖ를 생성하고, 각각의 모델 M₁~Mₖ를 훈련시킨다.

새로운 데이터 입력 시 각 모델이 예측하고, 투표(Classification) 또는 평균(Regression)으로 최종 출력을 결정한다.

09
Voting 메커니즘 & 앙상블이 효과적인 이유
🗳 투표 메커니즘
  • 각 분류기가 클래스 레이블을 출력 ("투표")
  • 분류: 다수결 투표(Majority Voting)
  • 회귀: 평균(Averaging)
원칙 앙상블은 다수의 모델이 틀릴 때에만 오류가 발생한다. → 단일 모델보다 더 강건하고 안정적이다.
🔗 모델 간 낮은 상관관계의 중요성
  • 모든 모델이 매우 유사하면 같은 실수를 저지르는 경향이 있다
  • 모델들이 덜 상관될수록 오류가 겹칠 가능성이 낮다
  • 따라서 다양한 훈련 세트나 알고리즘을 사용하여 다양성을 확보해야 한다
10
Bagging (Bootstrap Aggregating)
직관적 비유 의료 진단을 원할 때 한 명의 의사에게 묻는 대신, 여러 의사에게 상담하고 각자의 의견을 들은 후 다수결로 최종 결정을 내린다. 집단이 내린 결정이 단일 전문가의 결정보다 신뢰할 수 있다. (의사 → 분류기, 진단 → 예측)
📋 Bagging 기본 아이디어
  • d개 샘플의 데이터셋 D가 주어질 때
  • 여러 훈련 세트 D₁, D₂, …, Dₖ 생성
  • 각 데이터셋은 Bootstrap 샘플링(복원 추출)으로 생성
  • 일부 샘플은 여러 번 나타날 수 있고, 일부는 전혀 나타나지 않을 수 있다
⚙ 훈련 프로세스
  • 각 반복 i = 1, 2, …, k 에 대해:
    • D에서 복원 추출로 Dᵢ 샘플링
    • Dᵢ를 사용하여 모델 Mᵢ 훈련
  • 결과: 여러 모델 M₁, M₂, …, Mₖ 획득
  • 최종 예측: 다수결(분류) 또는 평균(회귀)
✅ Bagging이 효과적인 이유
1. 분산 감소 개별 모델은 크게 변동할 수 있지만, 여러 모델의 평균은 안정적이다.
2. 예측 안정화 여러 모델을 평균하면 예측의 변동이 줄어든다.
3. 노이즈·과적합 처리 다양한 훈련 샘플이 다양한 모델을 만들고, 집계를 통해 노이즈 영향이 감소한다.
📜 Bagging 알고리즘 (Pseudocode)
// Algorithm: Bagging
// Create an ensemble of classification models (equally weighted)

Input:  D (d training tuples), k (# models), learning scheme
Output: Composite model M*

Method:
(1) for i = 1 to k do  // create k models
(2)     create bootstrap sample D_i, by sampling D with replacement
(3)     use D_i and learning scheme to derive model M_i
(4) endfor

To classify tuple X:
    let each of the k models classify X and return the majority vote
11
Boosting & AdaBoost
직관적 비유 여러 의사에게 동등하게 묻는 Bagging과 달리, Boosting은 과거 성과를 기반으로 각 의사에게 다른 가중치를 부여한다. 더 신뢰할 수 있는 의사가 더 큰 영향력을 가진다. Boosting은 잘못 분류된(어려운) 샘플에 더 집중하면서 점진적으로 성능을 향상시킨다.
⚡ Boosting 기본 메커니즘
  • 모델을 순차적(Sequential)으로 구축
  • 각 새 모델은 이전 오류를 수정하는 데 집중
  • 모든 훈련 샘플에 동일 가중치로 초기화
  • 잘못 분류된 샘플의 가중치를 증가
  • k번 반복 → 모델이 점진적으로 어려운 예시에 집중
🧠 AdaBoost — 반복 학습
  • 초기화: 각 훈련 샘플에 동일 가중치 wᵢ = 1/d
  • 각 라운드에서: 가중치에 따라 데이터 샘플링 → Mᵢ 훈련 → 오류 계산
  • 잘못 분류: 가중치 증가
  • 올바르게 분류: 가중치 감소
  • 낮은 오류의 모델이 더 높은 가중치를 받음
📜 AdaBoost 알고리즘 (Pseudocode)
// Algorithm: AdaBoost (Boosting)
// Each classifier gives a weighted vote

Input:  D (d class-labeled tuples), k (# rounds), learning scheme
Output: Composite model

Method:
(1)  initialize weight of each tuple in D to 1/d
(2)  for i = 1 to k do
(3)      sample D with replacement according to weights → D_i
(4)      use D_i to derive model M_i
(5)      compute error(M_i)
(6)      if error(M_i) > 0.5 then
(7)          abort the loop
(8)      endif
(9)      for each correctly classified tuple in D do
(10)         multiply weight by error(M_i) / (1 - error(M_i))  // update weights
(11)     normalize weights
(12) endfor

To classify tuple X:
(1)  initialize weight of each class to 0
(2)  for i = 1 to k do
(3)      w_i = log( (1 - error(M_i)) / error(M_i) )  // classifier vote weight
(4)      c = M_i(X)  // get class prediction
(5)      add w_i to weight for class c
(6)  endfor
(7)  return class with largest weight
✅ Boosting이 효과적인 이유

📚 강의 핵심 정리

✂

Pruning (가지치기)

완전 성장 트리의 과적합 문제를 해결. Pre-pruning(조기 중단)과 Post-pruning(사후 제거)의 두 방법이 있으며, 모델 복잡성과 일반화의 균형이 핵심이다.

📊

모델 평가

훈련 정확도는 과적합으로 낙관적. Confusion Matrix 기반 지표(Precision, Recall, F1)를 사용하고, CV나 Bootstrap으로 신뢰할 수 있는 평가를 수행한다.

⚖

클래스 불균형

불균형 데이터에서 Accuracy는 오해를 줄 수 있다. Precision, Recall, F1-score를 사용하여 소수(Minority) 클래스 탐지 능력을 정확히 평가해야 한다.

🎭

앙상블 학습

여러 모델을 결합하여 단일 모델보다 강력한 예측기를 만든다. 분류는 다수결 투표, 회귀는 평균. 모델 간 낮은 상관관계가 핵심이다.

🎒

Bagging

Bootstrap 샘플링으로 여러 훈련 세트를 생성하고, 병렬로 모델을 훈련한다. 주로 분산(Variance)을 감소시키며, Random Forest가 대표 알고리즘이다.

⚡

Boosting & Random Forest

Boosting은 잘못 분류된 샘플에 가중치를 부여해 순차적으로 학습(편향 감소). Random Forest는 Bagging + 무작위 특성 선택으로 안정성과 정확도를 극대화한다.

방법핵심 아이디어주요 감소예측 방법대표 알고리즘
BaggingBootstrap + 병렬 훈련분산(Variance)다수결 / 평균Random Forest
Boosting순차 훈련, 가중치 조정편향(Bias)가중 투표AdaBoost, XGBoost
Random ForestBagging + 무작위 특성분산 + 상관관계다수결 / 평균sklearn RandomForest
