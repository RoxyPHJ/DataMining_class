Q01 ⭐ 슬라이드 5–6
조건부 확률 P(A|B)는 사건 B가 발생했을 때 사건 A가 발생할 확률이며, 공식은 P(A|B) = P(A∩B) / P(B) 이다. (단, P(B) > 0)
정답
O (참)
조건부 확률의 정의 그대로이다. 사건 B가 발생했다는 조건 아래에서 A가 발생할 확률은 두 사건의 교집합 확률을 B의 확률로 나눈 값이다.
P(A|B) = P(A∩B) / P(B), 단 P(B) > 0
예) 주사위에서 4 이상(B={4,5,6})이 나왔을 때 짝수(A={2,4,6})일 확률: P(A|B) = (2/6)/(3/6) = 2/3
Q02 ⭐ 슬라이드 7
연속 확률 변수는 확률 질량 함수(PMF)로 기술되며, 이산 확률 변수는 확률 밀도 함수(PDF)로 기술된다.
정답
X (거짓)
반대이다. 이산 확률 변수는 PMF(확률 질량 함수)로, 연속 확률 변수는 PDF(확률 밀도 함수)로 기술된다.
이산: P(X = xᵢ) ≥ 0, ΣP(X = xᵢ) = 1 → PMF 연속: f(x) ≥ 0, ∫f(x)dx = 1 → PDF
Q03 ⭐⭐ 슬라이드 13–14
MAP Classifier와 ML Classifier는 항상 동일한 결과를 출력한다.
정답
X (거짓)
ML Classifier는 MAP Classifier의 특수 케이스로, 모든 클래스의 사전확률이 동일(uniform prior)할 때만 두 분류기가 같은 결과를 낸다.
MAP: argmax p(x|wᵢ)·P(wᵢ) ML: argmax p(x|wᵢ) ← P(wᵢ)가 동일하면 약분 가능
사전확률이 다를 경우 MAP는 P(wᵢ)를 가중치로 반영하므로 결과가 달라질 수 있다.
Q04 ⭐⭐ [응용] 슬라이드 17–19
최소 위험 베이지안 분류기에서 c₁₂(실제 w₁을 w₂로 오분류 비용)가 c₂₁(실제 w₂를 w₁로 오분류 비용)보다 매우 크다면, 결정 경계는 w₂ 쪽으로 이동하여 w₁으로 분류되는 영역이 넓어진다.
정답
O (참)
조건 변형 적용: c₁₂ > c₂₁인 경우를 우도비 결정 규칙으로 분석한다.
우도비 임계값 = (c₂₁ - c₂₂)·P(w₂) / (c₁₂ - c₁₁)·P(w₁)
c₁₂가 커질수록 분모가 커져 임계값이 작아진다. 즉, 더 작은 우도비로도 w₁으로 분류되어 w₁ 결정 영역이 넓어진다.
직관적으로: w₁을 w₂로 오분류하는 것이 비용이 매우 크면, 가능한 한 w₁으로 분류하려 한다.
Q05 ⭐⭐ [응용] 슬라이드 16
두 클래스의 분포가 전혀 겹치지 않더라도 (완전히 분리된 경우), 분류기의 오류 확률은 항상 0보다 크다.
정답
X (거짓)
조건 변형 적용: 강의에서는 "겹침이 클수록 오류가 크다"고 했는데, 역방향으로 생각하면 된다.
E = (1/2)∫₋∞ᵗ p(x|w₂)dx + (1/2)∫ₜ∞ p(x|w₁)dx
두 분포가 완전히 분리된 경우, 겹치는 영역의 확률 질량이 0이므로 이론적으로 오류 확률 E = 0이 가능하다. 강의의 핵심은 "실제로는 분포가 겹쳐 오류가 불가피하다"는 것이지, 항상 E > 0이라는 뜻이 아니다.
Q06 ⭐⭐⭐ [응용] 슬라이드 8, 20–21
나이브 베이즈 분류기에서 조건부 독립 가정이 실제로 성립하지 않으면, 분류 결과는 반드시 부정확해지므로 실용적으로 사용할 수 없다.
정답
X (거짓)
개념 결합: 독립 가정 + 나이브 베이즈 실용성을 함께 적용한다. 강의에서 명시적으로 "독립 가정이 비현실적이지만 실제로 잘 동작한다(performs well in practice)"고 언급한다.

이유: 분류에는 정확한 확률값이 아니라 클래스 간 상대적 비교만 필요하다. 독립 가정이 틀려도 상대적 순위(argmax)가 올바르게 유지되는 경우가 많다.

Q07 ⭐ 슬라이드 9
베이즈 정리 P(X|Y) = P(Y|X)·P(X) / P(Y) 에서 P(Y)를 무엇이라 부르는가? 또한 P(Y)의 전개식을 쓰시오.
정답
주변확률 (Marginal Probability)
P(Y)는 주변확률(marginal probability) 또는 정규화 상수(normalization constant)라 부른다.
P(Y) = Σᵢ P(Y|Xᵢ) · P(Xᵢ) (모든 가능한 Xᵢ에 대한 합산)
분류 문제에서는 P(Y)가 모든 클래스에 대해 동일하므로, 사후확률 비교 시 생략 가능하다 (MAP 분류기의 근거).
Q08 ⭐⭐ 슬라이드 9–10
강의의 가방 예제에서 가방 A(흰 공 7개, 파란 공 3개), 가방 B(흰 공 9개, 파란 공 6개)이고 각 가방을 선택할 확률이 동일하다(P(A)=P(B)=0.5).
흰 공을 뽑았을 때 가방 B에서 뽑혔을 사후확률 P(B|white)를 기약분수로 구하시오.
정답
6/13
P(white|B) = 9/15 = 3/5 = 0.6 P(A) = P(B) = 0.5 P(white) = P(white|A)·P(A) + P(white|B)·P(B) = 0.7×0.5 + 0.6×0.5 = 0.65 P(B|white) = P(white|B)·P(B) / P(white) = (0.6×0.5) / 0.65 = 0.3 / 0.65 = 6/13 ≈ 0.462
P(A|white) = 7/13 ≈ 0.538, P(B|white) = 6/13 ≈ 0.462이므로 흰 공을 뽑았을 때 가방 A에서 뽑혔을 가능성이 더 높다.
Q09 ⭐⭐ 슬라이드 13–14
MAP(Maximum A Posteriori) Classifier의 결정 규칙을 수식으로 쓰시오. (argmax 형태로, 사용하는 확률 요소를 모두 포함할 것)
정답
argmax p(x|wᵢ)·P(wᵢ)
MAP 분류기는 사후확률 비교에서 분모 p(x)를 제거하고, 우도와 사전확률의 곱으로 결정한다.
P(wᵢ|x) = p(x|wᵢ)·P(wᵢ) / p(x) → p(x)는 모든 클래스에 동일 → 제거 → argmax p(x|wᵢ)·P(wᵢ)
이것이 Posterior Classifier의 근사적 표현이며, ML Classifier는 P(wᵢ)가 균일할 때의 특수 케이스이다.
Q10 ⭐⭐ [응용] 슬라이드 9–10
[수치 변형] 가방 A에 흰 공 6개·파란 공 4개, 가방 B에 흰 공 8개·파란 공 2개가 있다. 각 가방을 선택할 확률은 동일(P(A)=P(B)=0.5)하다.
흰 공을 뽑았을 때 P(A|white)를 기약분수로 구하시오.
정답
3/7
풀이 과정: 베이즈 정리를 동일하게 적용하되 수치만 변경한다.
P(white|A) = 6/10 = 0.6 P(white|B) = 8/10 = 0.8 P(white) = 0.6×0.5 + 0.8×0.5 = 0.7 P(A|white) = P(white|A)·P(A) / P(white) = (0.6 × 0.5) / 0.7 = 0.3 / 0.7 = 3/7 ≈ 0.429
흰 공이 많은 B 쪽으로 사후확률이 더 많이 쏠려 P(B|white) = 4/7 ≈ 0.571이 된다.
Q11 ⭐⭐⭐ [응용] 슬라이드 17–19
최소 위험 분류기에서 c₁₁ = 0, c₂₂ = 0, c₁₂ = 2, c₂₁ = 1이고 P(w₁) = P(w₂) = 0.5일 때, 우도비 결정 규칙의 임계값(threshold)은 얼마인가? 즉, p(x|w₁)/p(x|w₂) > ? 이면 w₁으로 분류한다.
정답
1/2 (= 0.5)
풀이 과정: 우도비 임계값 공식에 대입한다.
임계값 = (c₂₁ - c₂₂)·P(w₂) / (c₁₂ - c₁₁)·P(w₁) = (1 - 0)×0.5 / (2 - 0)×0.5 = 0.5 / 1.0 = 1/2
따라서 p(x|w₁)/p(x|w₂) > 1/2 이면 w₁으로 분류한다.
c₁₂=2로 c₂₁=1보다 크므로 임계값이 1보다 작아져(0.5), w₁으로 분류되는 영역이 더 넓어진다.
Q12 ⭐⭐⭐ [응용] 슬라이드 9, 22
[역방향] 어떤 이메일 데이터셋에서 P(spam) = 0.4, P(ham) = 0.6이다. 특정 단어 w가 포함된 이메일의 사후확률이 P(spam|w) = P(ham|w)로 동일하다면, 우도비 P(w|spam) / P(w|ham)의 값은 얼마인가?
정답
3/2 (= 1.5)
역방향 적용: 결과(사후확률 동일)에서 입력(우도비)을 역산한다.
P(spam|w) = P(ham|w) 조건에서: P(w|spam)·P(spam) / P(w) = P(w|ham)·P(ham) / P(w) → P(w|spam)·P(spam) = P(w|ham)·P(ham) → P(w|spam) / P(w|ham) = P(ham) / P(spam) = 0.6 / 0.4 = 3/2
직관: 스팸 사전확률이 낮을수록(P(spam)=0.4 < 0.6), 사후확률을 동일하게 만들려면 우도비가 더 높아야 한다.

Q13 ⭐⭐ 슬라이드 21–22
아래는 나이브 베이즈 스팸 필터를 Python으로 구현한 코드이다. 빈칸 ①②③을 채우시오.
def naive_bayes_spam(email_words, word_counts, n_total, n_spam): # ① 사전확률 P(spam) p_spam = n_spam / # ② 우도 곱 계산 (나이브 가정) log_likelihood = sum( log(word_counts[w] / n_spam) for w in email_words ) # ③ 사후확률 ∝ 사전확률 × 우도 곱 log_posterior = log() + return log_posterior
정답
① n_total    ② p_spam    ③ log_likelihood
① P(spam) = n_spam / n_total → 전체 이메일 수로 나눔 ② log(P(spam)) → 나이브 베이즈에서 log 변환으로 곱셈→덧셈 ③ log_likelihood → Σ log P(wᵢ|spam)의 합산값
로그 공간에서의 나이브 베이즈: log P(spam|w₁,...,wₙ) ∝ log P(spam) + Σ log P(wᵢ|spam)
로그를 사용하는 이유: 매우 작은 확률값들을 곱하면 언더플로우가 발생하므로, 로그 덧셈으로 변환한다.
Q14 ⭐⭐ 슬라이드 13–14
아래는 이진 MAP 분류기를 구현한 코드이다. 빈칸 ①②를 채우시오.
def map_classifier(x, likelihood_w1, likelihood_w2, prior_w1, prior_w2): # MAP: argmax p(x|wᵢ) * P(wᵢ) score_w1 = likelihood_w1 * score_w2 = likelihood_w2 * if score_w1 > score_w2: return "w1" else: return "w2"
정답
① prior_w1    ② prior_w2
MAP 분류기의 결정 규칙: 우도(likelihood)와 사전확률(prior)의 곱이 가장 큰 클래스를 선택한다.
score_w1 = p(x|w₁) × P(w₁) = likelihood_w1 × prior_w1 score_w2 = p(x|w₂) × P(w₂) = likelihood_w2 × prior_w2 if score_w1 > score_w2: → w₁으로 분류
만약 prior_w1 == prior_w2라면 사전확률이 약분되어 ML Classifier와 동일한 결과가 된다.
Q15 ⭐⭐ [응용] 슬라이드 5–6
[수치 변형] 아래 코드는 주사위 실험에서 조건부 확률을 계산한다. 사건 A = {2, 4, 6} (짝수), 사건 B = {3, 4, 5, 6} (3 이상)일 때 P(A|B)를 구하도록 빈칸 ①②를 채우시오.
sample_space = {1, 2, 3, 4, 5, 6} A = {2, 4, 6} B = {3, 4, 5, 6} # 강의 예제와 다른 B (수치 변형) A_inter_B = # A ∩ B p_A_given_B = len(A_inter_B) / # 기댓값: p_A_given_B = ?
정답
① A & B 또는 {4,6}    ② len(B)    결과: P(A|B) = 2/4 = 0.5
수치 변형 풀이: 강의 예제에서 B={4,5,6}이었지만, 여기서는 B={3,4,5,6}으로 확장된다.
A ∩ B = {2,4,6} ∩ {3,4,5,6} = {4,6} → len = 2 P(B) = 4/6 → len(B) = 4 P(A|B) = P(A∩B)/P(B) = (2/6)/(4/6) = 2/4 = 1/2 = 0.5
강의 예제(B={4,5,6})와 비교: P(A|B) = 2/3 → 0.667에서, B가 더 넓어지면(하한이 3으로 낮아짐) P(A|B)가 0.5로 감소. B에 홀수 3이 추가되었기 때문.
Q16 ⭐⭐⭐ [응용] 슬라이드 17–19
[개념 결합] 아래는 최소 위험 분류기의 결정 함수이다. 빈칸 ①②③을 채우시오.
(q₁ < q₂ 이면 w₁으로 분류, q₁ > q₂ 이면 w₂로 분류)
def min_risk_classifier(px_w1, px_w2, pw1, pw2, c11, c12, c21, c22): # q₁: x를 w₁으로 결정할 때의 조건부 위험 q1 = c11 * + c21 * # q₂: x를 w₂으로 결정할 때의 조건부 위험 q2 = c12 * px_w1 * pw1 + return "w1" if q1 < q2 else "w2"
정답
① px_w1 * pw1    ② px_w2 * pw2    ③ c22 * px_w2 * pw2
q₁ = c₁₁·p(x|w₁)·P(w₁) + c₂₁·p(x|w₂)·P(w₂) = c11*(px_w1*pw1) + c21*(px_w2*pw2) q₂ = c₁₂·p(x|w₁)·P(w₁) + c₂₂·p(x|w₂)·P(w₂) = c12*(px_w1*pw1) + c22*(px_w2*pw2)
q₁이 작을수록 "w₁으로 결정"하는 것이 기대 위험이 낮다. c₁₁=c₂₂=0(올바른 분류 비용 없음), c₁₂≠c₂₁이면 비대칭 손실 구조가 만들어진다.

Q17 ⭐⭐ 슬라이드 9
사전확률(Prior Probability)과 사후확률(Posterior Probability)의 차이를 설명하고, 베이즈 정리에서 두 확률이 어떻게 연결되는지 서술하시오.
모범 답안
사전확률 P(X): 어떠한 증거(데이터)도 관찰하기 전에 사건 X에 대해 가지는 초기 믿음(belief)의 확률이다. 과거 경험이나 도메인 지식으로 결정된다.

사후확률 P(X|Y): 새로운 증거 Y를 관찰한 후 업데이트된 X에 대한 확률이다. 사전확률에 우도 정보를 반영하여 갱신된 값이다.

베이즈 정리에서의 연결:
P(X|Y) = P(Y|X) · P(X) / P(Y) ↑ ↑ ↑ 사후확률 우도 사전확률
즉, 새로운 증거 Y(우도 P(Y|X))를 통해 사전확률 P(X)를 사후확률 P(X|Y)로 업데이트하는 프레임워크이다. 가방 예제에서 P(A)=0.5(사전)가 흰 공을 뽑은 후 P(A|white)=7/13(사후)으로 바뀌는 것이 이에 해당한다.
Q18 ⭐⭐ 슬라이드 8, 20–22
나이브 베이즈(Naïve Bayes) 분류기에서 "나이브(Naïve)"한 이유는 무엇인가? 그 가정이 비현실적임에도 실용적으로 사용되는 이유를 함께 설명하시오.
모범 답안
나이브한 이유: 클래스 레이블이 주어졌을 때 모든 특징(feature)들이 조건부 독립(conditionally independent)이라 가정하기 때문이다.
P(w₁,w₂,...,wₙ|spam) = P(w₁|spam)·P(w₂|spam)·...·P(wₙ|spam)
실제로는 단어 "free"와 "discount"가 스팸 메일에서 함께 자주 등장하는 등 특징 간 상관이 존재하므로 이 가정은 대부분 현실에서 성립하지 않는다.

그럼에도 실용적인 이유:
분류 문제에서는 정확한 확률값보다 클래스 간 상대적 크기(argmax)가 중요하다. 독립 가정이 틀리더라도 클래스 순위가 올바르게 유지되는 경우가 많아 실제 성능이 좋다. 또한 파라미터 수가 적어 학습에 필요한 데이터양이 적고 계산이 빠르다.
Q19 ⭐⭐⭐ [응용] 슬라이드 13–15
[조건 변형] 아래 세 상황에서 Posterior, MAP, ML 분류기 중 어느 분류기를 사용해야 하는지 각각 쓰고, 그 이유를 서술하시오.

(가) 클래스별 사전확률을 정확히 알고 있으며, 사후확률을 직접 계산할 수 있다.
(나) 우도 함수는 알지만 모든 클래스의 사전확률이 같다.
(다) 우도 함수는 알고, 사전확률도 알지만 사후확률 계산이 어렵다 (p(x) 계산 불가).
모범 답안
(가) → Posterior (Bayes) Classifier
사후확률 P(wᵢ|x)를 직접 계산할 수 있으므로 argmax P(wᵢ|x)를 사용. 이론적으로 가장 최적이다.

(나) → ML Classifier
사전확률이 균일하면 P(wᵢ)가 약분되어 우도만으로 분류. argmax p(x|wᵢ). MAP와 ML이 동일한 결과를 낸다.

(다) → MAP Classifier
p(x) = Σ p(x|wᵢ)P(wᵢ)를 직접 계산하기 어렵더라도, 분류는 클래스 간 비교만 필요하므로 분모를 생략할 수 있다.
P(wᵢ|x) = p(x|wᵢ)·P(wᵢ) / p(x) → p(x)는 모든 클래스에 동일 → 제거 → argmax p(x|wᵢ)·P(wᵢ) ← MAP
Q20 ⭐⭐⭐ [응용] 슬라이드 17–19
[개념 결합] 의료 진단 시스템에서 질병(w₁) / 정상(w₂)을 분류한다. 아래 조건을 바탕으로 최소 위험 분류기의 손실 행렬을 설계하고, MAP 분류기 대비 결정 경계가 어느 방향으로 이동하는지 설명하시오.

조건: "질병인데 정상으로 분류"하는 것이 "정상인데 질병으로 분류"하는 것보다 10배 더 위험하다. 올바른 분류 비용은 0이다.
모범 답안
손실 행렬 설계:
결정: w₁(질병) 결정: w₂(정상) 실제: w₁(질병) c₁₁ = 0 c₁₂ = 10 ← 미탐지: 매우 위험 실제: w₂(정상) c₂₁ = 1 c₂₂ = 0 ← 거짓 경보: 덜 위험
우도비 임계값 계산 (P(w₁)=P(w₂)=0.5 가정):
임계값 = (c₂₁ - c₂₂)·P(w₂) / (c₁₂ - c₁₁)·P(w₁) = (1 - 0)×0.5 / (10 - 0)×0.5 = 0.5 / 5.0 = 0.1
결정 경계 이동 방향:
MAP 분류기의 임계값은 P(w₁)=P(w₂)일 때 1.0이다. 최소 위험 분류기의 임계값은 0.1로 훨씬 작다.
즉, p(x|w₁)/p(x|w₂) > 0.1만 되어도 "질병(w₁)"으로 분류하게 된다.

→ 결정 경계가 w₂(정상) 방향으로 이동하여 w₁으로 분류되는 영역이 대폭 넓어진다. 이는 미탐지 비용이 매우 크기 때문에 조금이라도 질병 가능성이 있으면 질병으로 분류하려는 보수적 설계를 반영한다.
