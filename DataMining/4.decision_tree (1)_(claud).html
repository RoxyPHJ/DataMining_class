What is Classification

Slide 2/16
Definition
  • 데이터 분석 태스크로서 모델(분류기)을 학습
  • 범주형(이산형) 클래스 레이블을 예측
Examples
  • 대출 승인: safe vs. risky
  • 의료 진단: disease vs. normal
  • 스팸 탐지: spam vs. not spam
  • 자율 주행: 교통 표지판 인식

Classification vs. Prediction

Slide 3/16
Classification (분류)
  • 출력: 범주형 레이블
  • 예: yes/no, pos./neg., A/B/C
  • 목표: 이산 클래스 예측
Numerical Prediction (수치 예측)
  • 출력: 연속 값
  • 예: 집값, 소득, 판매량
  • 목표: 연속 값 예측 (회귀)
핵심 차이: Classification → Discrete / Regression → Continuous

Supervised Learning Setup

What is a Decision Tree?

Slide 6/16
Definition
  • 분류·회귀를 위한 지도학습 방법
  • 입력 x ∈ ℝᵈ로부터 출력 y 예측
  • 일련의 간단한 decision rules 사용
Key Idea
  • 일련의 질문을 순차적으로 묻는 구조
  • if-then rules의 집합 구성
  • 계층적 결정을 통해 입력을 출력에 매핑
Advantages
  • 도메인 지식이나 파라미터 튜닝 불필요
  • 다차원 데이터 처리 가능
  • 인간이 이해하기 직관적
  • 계산 효율적이며 성능 우수
Limitations
  • 훈련 데이터에 과적합(overfit) 가능성
  • 일부 분기가 노이즈·이상값 반영 가능

Decision Tree Induction

Slide 7/16
트리 구조
  • 플로우차트 형태의 구조
  • Internal Node: 속성에 대한 테스트
  • Branch: 테스트 결과에 해당하는 경로
  • Leaf Node: 클래스 레이블 포함
  • Root Node: 트리의 최상단 노드
분류 과정
  • 1. Root node에서 시작
  • 2. 각 노드에서 test condition 평가
  • 3. feature 값에 따라 해당 branch 이동
  • 4. leaf node 도달까지 반복
  • 5. leaf node의 레이블 = 예측 클래스
분류 = Root → Leaf 경로 탐색 (path traversal)
분류 과정 플로우

Representative Algorithms

Slide 8/16
ID3
분할 기준: Information Gain
속성 타입: 범주형만
가지치기: 없음
트리 형태: 다지(Multi-way)
Quinlan 1986
C4.5
분할 기준: Gain Ratio
속성 타입: 범주형 + 연속형
가지치기: Post-pruning
편향 감소: Yes
Quinlan 1993 (ID3 확장)
CART
분할 기준: Gini Index / MSE
트리 형태: Binary Tree 전용
분류+회귀: 모두 지원
가지치기: 포함
Breiman et al. 1984
공통 특징 (Common Characteristics)
  • Greedy (non-backtracking) 전략
  • Top-down 재귀적 트리 구성
  • 각 노드에서 순수한(pure) 분할 목표

Basic Algorithm

Slide 9/16
Generate_decision_tree(D, attribute_list)
Input
  • D: 훈련 튜플 + 클래스 레이블 집합
  • attribute_list: 후보 속성 집합
  • Attribute_selection_method: 분할 기준 결정 절차 (e.g., info gain, gini)
Output
  • Decision Tree
1create node N;
2if tuples in D are all same class C then
3return N // leaf node labeled C
4if attribute_list is empty then
5return N // majority voting
6apply Attribute_selection_method(D, attribute_list) // best split
7label N with splitting_criterion;
8if splitting_attribute is discrete-valued and multiway splits allowed then
9attribute_list ← attribute_list − splitting_attribute;
10for each outcome j of splitting_criterion:
11let D_j = tuples in D satisfying outcome j;
12if D_j is empty then
13attach leaf with majority class in D;
14else attach Generate_decision_tree(D_j, attribute_list); // 재귀
15return N;

Partitioning

Slide 10/16
(a) DISCRETE · MULTI-WAY SPLIT

속성의 각 값에 대해 별도 branch 생성. 범주형 속성에 적용.

(b) CONTINUOUS · THRESHOLD SPLIT

임계값 기준 ≤ 와 > 두 branch로 분할. 연속형 속성에 적용.

(c) BINARY SPLIT ON DISCRETE

범주형 속성을 두 부분집합으로 이진 분할. CART에서 사용.

Attribute Selection Measures

Slide 11–12/16
개념
  • 각 노드에서 최적 분할 속성 선택 휴리스틱
  • 데이터셋을 더 작은 부분집합으로 분할
  • 목표: 순수한(pure) 부분집합 생성
주요 측도
  • Entropy: 데이터셋 D의 불순도
  • Information Gain (ID3): 분할 후 불확실성 감소량
  • Gain Ratio (C4.5): 정규화된 정보 이득
  • Gini Index (CART): 데이터셋 불순도 측정
수식 정리
Entropy: Info(D) = −Σᵢ pᵢ · log₂(pᵢ)
where pᵢ = |Cᵢ,D| / |D|
Cond. Entropy: Info_A(D) = Σⱼ (|Dⱼ|/|D|) × Info(Dⱼ)
Info Gain: Gain(A) = Info(D) − Info_A(D)
Split Info: SplitInfo_A(D) = −Σⱼ (|Dⱼ|/|D|) · log₂(|Dⱼ|/|D|)
Gain Ratio: GainR(A) = Gain(A) / SplitInfo_A(D)
Gini(D): 1 − Σᵢ pᵢ²   where pᵢ = |Cᵢ,D| / |D|

Slide 13–14/16
Step 1: Entropy of D
Info(D) = −(9/14)·log₂(9/14) − (5/14)·log₂(5/14)
= 0.940 bits
Step 2: age 속성 분할
ageyesno

Step 3: 각 파티션 엔트로피
Info(youth) = −(2/5)·log₂(2/5) − (3/5)·log₂(3/5)
Info(middle_aged) = −(4/4)·log₂(4/4) = 0 (pure!)
Info(senior) = −(3/5)·log₂(3/5) − (2/5)·log₂(2/5)

Step 5: Information Gain
Gain(age) = 0.940 − 0.694 = 0.246 bits

Gain Ratio

Slide 15/16
Information Gain의 한계
  • 값이 많은 속성에 편향(bias) 발생
  • 예: product_ID (고유 식별자)
  • 각 값이 별도 파티션 → entropy = 0 → gain = Info(D) = 최대
  • 그러나 이는 실제로 유용한 분할이 아님
Gain Ratio (C4.5)
  • 분할 정보(Split Information)로 gain을 정규화
  • 데이터가 얼마나 고르게 분할되는지 측정
  • 클래스 기반이 아닌 분할 방법 기반
income 속성 예시
  • high: 4개, medium: 6개, low: 4개
SplitInfo_income(D)
= −(4/14)·log₂(4/14) − (6/14)·log₂(6/14) − (4/14)·log₂(4/14)
= 1.557
GainRatio(income) = Gain(income) / SplitInfo(income)
= 0.029 / 1.557 ≈ 0.019
Gain Ratio는 단순 정보 이득보다 더 나은 일반화 성능 제공

Gini Impurity

Slide 16/16
Gini Impurity란?
  • 데이터셋 D의 불순도(impurity) 측정
  • Gini = 0 → 완전히 순수 (단일 클래스)
  • Gini ↑ → 클래스 혼합도 증가
Gini(D) = 1 − Σᵢ pᵢ²
직관적 해석
  • 클래스 분포에 따라 무작위 레이블 선택 시 오분류 확률
  • 예: yes=0.7, no=0.3이면
  • 정확 예측 확률 = 0.7² + 0.3² = 0.58
  • 오분류 확률 = 1 − 0.58 = 0.42
계산 예시 (income 속성)
Gini(D) = 1 − (9/14)² − (5/14)² = 0.459
D₁: low+medium (10 tuples: 7yes, 3no)
D₂: high (4 tuples: 2yes, 2no)
Gini_split = (10/14)·Gini(D₁) + (4/14)·Gini(D₂)
= 0.443
분할 전 0.459 → 분할 후 0.443: 불순도 감소 → 더 나은 파티션

핵심 요약

Decision Tree (1) — 강의 핵심 정리
분류(Classification) 개념
  • 범주형 레이블 예측 (vs 회귀: 연속값 예측)
  • 지도학습: labeled 데이터로 f: x → y 학습
  • 3단계: 모델 구축 → 평가 → 예측
Decision Tree 구조
  • Root Node / Internal Node / Branch / Leaf Node
  • 분류 = Root에서 Leaf까지 경로 탐색
  • 귀납(Induction): 훈련 데이터로부터 재귀적 구성
대표 알고리즘 비교
  • ID3: 정보이득, 범주형만, 가지치기 없음
  • C4.5: 이득비율, 연속형 지원, 사후 가지치기
  • CART: 지니지수, 이진 트리, 분류+회귀
속성 선택 측도 공식
Entropy: −Σᵢ pᵢ·log₂(pᵢ)
Info Gain: Info(D) − Info_A(D)
Gain Ratio: Gain(A) / SplitInfo_A(D)
Gini: 1 − Σᵢ pᵢ²
예시 (14-tuple 데이터셋)
  • Info(D) = 0.940 bits (yes:9, no:5)
  • Gain(age) = 0.246 — 가장 높음 → Root 선택
  • middle_aged: 4yes/0no → entropy=0 → Leaf
  • SplitInfo(income) = 1.557 → GainR = 0.019
  • Gini(D) = 0.459 → 분할 후 0.443으로 감소
주의사항
  • 과적합(Overfitting) 위험 → 가지치기(Pruning) 필요
  • Info Gain: 값 많은 속성에 편향 → Gain Ratio로 보완
