01
4.1.1 회선 부호화 (Line Coding)
정의

일련의 2진 비트 데이터를 디지털 신호로 바꾸는 작업이다. 송신측에서 부호화(Line Coding), 수신측에서 복호화(Decoding)가 이루어진다.

핵심 흐름

디지털 데이터 (비트열) → 회선 부호화 → 디지털 신호 → 복호화 → 디지털 데이터

신호 요소 vs 데이터 요소
구분정의단위
신호 요소디지털 신호의 가장 짧은 단위—
데이터 요소데이터를 나타내는 가장 작은 단위체비트(bit)
r = 데이터 요소 / 신호 요소
r: 신호 요소당 데이터 요소 수를 나타내는 변수
데이터 전송률 vs 신호 전송률
데이터 전송률 N (Data Rate)

1초당 전송된 데이터 요소의 개수

단위: bps (bits per second)

= 비트율(Bit Rate)

신호 전송률 S (Signal Rate)

1초당 전송된 신호 요소의 개수

단위: baud

= 펄스율 = 변조율 = 보오율

S = N / r
💡 핵심 의미

r이 클수록 하나의 신호로 더 많은 비트를 전송 → 같은 비트율에서 신호 전송률이 낮아짐 = 대역폭 효율적

회선 부호화의 문제점
⚠️ 기준선 표류

수신 신호의 평균을 기준으로 데이터를 결정함

지속적인 0 또는 1은 기준선을 이동시켜 복호화 오류 유발

⚠️ 직류 성분

저주파 성분은 전선/변압기를 통과하지 못함

직류(DC) 성분 제거가 필요

⚠️ 자기 동기화

송신자와 수신자의 비트 간격이 일치해야 함

불일치 시 복호화 오류 발생

02
4.1.2 회선 부호화 방식
전체 분류 체계
극형 (Polar) 개요
  • 양과 음의 두 가지 전압 준위를 사용
  • 회선의 평균 전압 준위를 낮춤 → 직류 성분 문제 완화
  • 세부 방식: NRZRZ맨체스터차분 맨체스터
NRZ (Non-Return-to-Zero)

비영복귀: 비트 간격 내에서 전압이 0으로 돌아가지 않음


NRZ-L (Level)

  • 양전압 (+V): 비트 0
  • 음전압 (−V): 비트 1
  • 전압 준위가 비트값을 결정

NRZ-I (Invert)

  • 비트 1: 전환 발생
  • 비트 0: 전환 없음
  • 전환 여부가 비트값을 결정
단점

연속 0 또는 1 시 동기화 문제 (기준선 표류)

맨체스터 & 차분 맨체스터

Biphase 계열 — 비트 간격 중간에서 반드시 전환 발생


맨체스터 (Manchester)

  • 각 비트 간격 중간에 반전
  • 낮→높: 비트 0
  • 높→낮: 비트 1
  • 중간 전환이 동기화와 데이터 모두 담당

차분 맨체스터 (Differential Manchester)

  • 중간 반전: 항상 발생 (동기화 전용)
  • 시작점 전환 있음: 비트 0
  • 시작점 전환 없음: 비트 1
장점

중간 전환으로 자기 동기화 해결, DC 성분 없음

NRZ vs 맨체스터 vs 차분 맨체스터 비교
방식비트 결정 방법동기화DC 성분대역폭
NRZ-L 전압 준위 ❌ 취약 ❌ 있음 낮음 (효율적)
NRZ-I 전환 여부 △ 1에서만 △ 완화 낮음
맨체스터 중간 전환 방향 ✅ 우수 ✅ 없음 높음 (2배)
차분 맨체스터 시작점 전환 여부 ✅ 우수 ✅ 없음 높음 (2배)
다준위 방식 — 4D-PAM5

Four-Dimensional Five-Level Pulse Amplitude Modulation

특징

  • 4D: 데이터가 4개의 회선으로 동시 전송
  • 5준위: −2, −1, 0, +1, +2
  • 1G bps LAN에 사용 (기가비트 이더넷)
준위 0의 역할

0 준위는 오류 검출에 사용

03
4.2 아날로그→디지털 변환 (PCM)
PCM (Pulse Code Modulation) 개요

아날로그 신호를 디지털 데이터로 변환하기 위해 가장 널리 사용되는 기법

3단계 변환 과정

아날로그 신호 → [1단계: 표본화] → [2단계: 양자화] → [3단계: 부호화] → 디지털 데이터

1단계: 표본화 (Sampling)

일정 간격마다 아날로그 신호의 진폭을 측정하여 펄스를 생성


🔑 나이퀴스트 정리 (Nyquist Theorem)

표본화율은 신호에 포함된 최대 주파수의 최소 2배이어야 한다

fs ≥ 2 × fmax

📞 예제 (전화 음성)

최대 주파수: 4,000 Hz

→ 표본화율: 8,000 samples/sec

2단계: 양자화 (Quantizing)

표본화된 값에 특정 구간에 속하는 정수값을 할당하는 방법


양자화 과정

  1. 신호 범위 Vmin ~ Vmax 를 L개 구간으로 분할
  2. 각 구간의 높이: Δ (델타)
  3. 각 구간 중간점에 0 ~ L-1 값 할당
  4. 표본값을 가장 가까운 양자화 값으로 근사
⚠️ 양자화 잡음

근사 과정에서 발생하는 오차
L이 클수록(구간이 세밀) 잡음 감소

3단계: 부호화 (Encoding)

각 표본이 nb 비트의 부호로 변환됨

Bit Rate = fs × nb
표본화율 × 표본당 비트 수
📝 예제 4.14 — 음성 디지털화

인간 목소리 주파수 범위: 0 ~ 4,000 Hz

나이퀴스트에 의해 표본화율: fs = 8,000 samples/sec

표본당 비트 수: nb = 8 bits

Bit Rate = 8,000 × 8 = 64,000 bps = 64 Kbps
PCM 복호기 구성요소

PCM 디코더: 수신된 디지털 데이터를 역순으로 처리하여 아날로그 신호를 재생

디지털 → 복호화 → 역양자화 → 재구성 → 아날로그 신호

4.3 전송 방식
병렬 전송 (Parallel Transmission)

한 번에 n개 비트를 동시에 전송, n개의 전선 사용


✅ 장점

직렬 전송에 비해 n배 빠른 전송 속도


❌ 단점

가격이 비쌈
비용으로 인해 짧은 거리로 제한


사용 예 프린터, 컴퓨터 내부 버스

직렬 전송 (Serial Transmission)

한 비트가 다른 비트 뒤에 오므로 하나의 채널만 필요


✅ 장점

병렬 대비 1/n 비용 절감
하나의 통신 채널만 필요


내부 통신: 병렬 구조

→ 인터페이스에서 변환 장치 필요

  • 송신측: 병렬 → 직렬
  • 수신측: 직렬 → 병렬

구성 방식

비동기식 또는 동기식으로 구성

비동기식 vs 동기식 직렬 전송
비동기식 (Asynchronous)
  • 각 바이트 앞에 시작 비트(0) 추가
  • 바이트 끝에 정지 비트(1) 추가
  • 총 8 데이터 + 2 추가 비트 구성
  • 수신기는 각 바이트마다 재동기화
동기식 (Synchronous)
  • 프레임으로 구성된 다수의 바이트
  • 바이트 사이 간격 없음
  • 수신자가 8비트 단위로 그룹 분리
  • 비동기식보다 고속, 고속 응용에 유리
  • 동기화는 데이터 링크 계층에서 수행

비동기 프레임 구조

전송 방식 종합 비교
구분병렬 전송직렬-비동기직렬-동기
채널 수 n개 (동시) 1개 1개
속도 가장 빠름 보통 빠름
비용 비쌈 저렴 저렴
거리 단거리 제한 없음 제한 없음
동기화 — 바이트마다 재동기화 데이터 링크 계층
오버헤드 없음 2비트/바이트 프레임 헤더
응용 내부 버스, 프린터 모뎀, RS-232 고속 네트워크

🎯 시험 포인트

  • 맨체스터: 중간 전환이 비트+동기화 담당
  • 차분 맨체스터: 시작점 전환→0, 무변화→1
  • NRZ 문제: 기준선 표류, DC 성분
  • 양자화 잡음: 근사 오차 (L↑ → 잡음↓)
  • 비동기: 시작비트(0) + 정지비트(1)
  • 동기식: 간격 없음, 데이터링크 계층 동기화
  • 4D-PAM5: 5준위(-2,-1,0,1,2), 1G LAN
