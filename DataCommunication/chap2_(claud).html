01

프로토콜 계층화

개념

복잡한 네트워크 통신 기능을 여러 계층(layer)으로 분리하여 각각 독립적으로 설계·구현하는 방식입니다.

핵심: 각 계층은 바로 아래 계층의 서비스를 이용하고, 바로 위 계층에게 서비스를 제공합니다.
✅ 장점
  • 서비스 구현을 각 기능별로 분리
  • 중간 시스템을 단순화
  • 계층별 독립적인 개발·유지보수 가능
  • 특정 계층만 교체·업그레이드 가능
❌ 단점
  • 계층 간 상호 연관이 있는 경우 정보 교환이 어려움
  • 계층 간 중복 기능 발생 가능
  • 추가 오버헤드(헤더) 발생

OSI 모델 개요

OSI 모델이란?

Open Systems Interconnection — 모든 유형의 컴퓨터 시스템 간의 통신을 허용하는 네트워크 시스템 설계를 위한 계층 구조입니다.

역사: TCP/IP가 많은 시간과 돈을 들여 완전히 자리잡은 후 OSI가 완성됨 → 일부 계층은 완전히 정의되지 않아 실무 보급 실패

03

OSI 7계층 상세

① 물리층 (Physical Layer) Layer 1PDU: Bit

물리적 매체를 통한 비트 스트림 전송에 요구되는 기능을 담당합니다. 기계적·전기적 특성(케이블, 커넥터)을 다룹니다.

  • 데이터 링크층으로부터 받은 데이터를 통신 링크로 전송 가능한 형태로 변환
  • 비트 스트림을 전자기 신호로 변환
  • 매체를 통한 신호 전송 감독
  • 물리적 장치·인터페이스가 전송에 필요한 기능과 처리 절차 규정
② 데이터 링크층 (Data Link Layer) Layer 2PDU: Frame

노드 대 노드 전달(node-to-node delivery)의 책임을 가집니다. 상위 계층(네트워크층)에서 오류 없는 물리층처럼 보이도록 합니다.

  • 프레임 구성: 네트워크층으로부터 받은 비트 스트림을 프레임 단위로 나눔
  • 물리 주소 지정: 송신자·수신자의 물리 주소(MAC)를 헤더에 추가
  • 흐름 제어: 수신자의 수신 데이터 전송률을 고려하여 전송 제어
  • 오류 제어: 손상·손실된 프레임 발견 및 재전송 (트레일러 활용)
  • 접근 제어: 한 순간에 하나의 장치만 동작하도록 제어
전달 범위: 홉-대-홉(hop-to-hop), 노드-대-노드(node-to-node) — 인접 노드까지만
③ 네트워크층 (Network Layer) Layer 3PDU: Packet

패킷을 발신지-대-목적지(source-to-destination) 전달에 대한 책임을 가집니다.

  • 발신지-대-목적지 전달: 데이터링크층과 달리 최종 목적지까지 전달
  • 논리 주소 지정(Logical Addressing): 패킷 헤더에 발신지·목적지 논리 주소(IP) 추가
  • 라우팅(Routing): 패킷이 최종 목적지에 전달될 수 있도록 경로 지정 및 교환
비교: 데이터링크층은 노드간 전달, 네트워크층은 발신지-목적지 간 전달
④ 전송층 (Transport Layer) Layer 4PDU: Segment

전체 메시지의 프로세스 대 프로세스 전달에 대한 책임을 가집니다. 전체 메시지가 완전하게 바른 순서로 도착하는 것을 보장합니다.

  • 포트 주소 지정: 포트 주소 포함 — 네트워크층이 컴퓨터까지, 전송층이 특정 프로세스까지 전달
  • 분할과 재조립(Segmentation & Reassembly): 세그먼트 단위로 분할, 순서 번호로 재조립·손실 판단
  • 연결 제어: 비연결(connectionless) 및 연결지향(connection-oriented)
  • 흐름 제어 / 오류 제어
⑤ 세션층 (Session Layer) Layer 5
  • 세션 관리(Session management)
  • 동기화(Synchronization)
  • 대화 제어(Dialog control)
  • 원활한 종료(Graceful Close)
⑥ 표현층 (Presentation Layer) Layer 6
  • 통신 장치 간 상호 운용성(interoperability) 보장
  • 구문(syntax)·의미(semantic) 처리
  • 데이터 암호화 / 복호화
  • 데이터 압축 / 확장
⑦ 응용층 (Application Layer) Layer 7

사용자가 네트워크에 접근할 수 있도록 하고 사용자 인터페이스를 제공합니다.

  • 원격 로그인
  • 파일 액세스, 전송, 관리(File access, transfer, and management)
  • 우편 서비스(Mail services)
  • WWW 접근(웹 접근)

TCP/IP 프로토콜

OSI vs TCP/IP 비교

TCP/IP 프로토콜에는 세션층과 표현층이 없습니다. OSI의 5, 6, 7층 기능이 TCP/IP의 응용층에 통합됩니다.

주소 지정

4쌍의 주소 체계

프로토콜 계층화와 관련하여 발신지와 목적지 간 통신을 위해 4쌍의 주소를 사용합니다.

계층 주소 유형 예시 특징
응용층 이름(Name) www.example.com 사람이 읽기 쉬운 문자 주소
전송층 포트 주소(Port Address) HTTP: 80, HTTPS: 443 16비트, 특정 프로세스(응용 프로그램) 식별
네트워크층 논리 주소(IP Address) 192.168.1.1 32비트(IPv4), 전체 인터넷에서 고유한 주소
데이터링크층 물리 주소(MAC Address) 00:1A:2B:3C:4D:5E 48비트, 로컬 네트워크 내 장치 식별
Figure — 계층별 헤더 추가 (캡슐화) 과정
포트 주소 (Port Address)

전송층에서 사용하는 16비트 주소로, 특정 컴퓨터의 특정 프로세스(응용 프로그램)를 식별합니다.

역할 분리: 네트워크층(IP) → 정확한 컴퓨터까지 / 전송층(포트) → 해당 컴퓨터의 정확한 프로세스까지

Switch vs Router

TCP/IP 계층적 구조 구성 요소

링크층 스위치를 가진 여러 LAN이 하나의 라우터에 연결되어 인터넷을 구성합니다.

주의: 여기서 switch는 링크계층에 한정된 이더넷 스위치를 의미합니다.

핵심 요약

프로토콜 계층화

기능별 분리로 단순화·유지보수 용이. 계층 간 정보 교환 어려움이 단점.

OSI 7계층

물리→링크→네트워크→전송→세션→표현→응용. PDU: Bit→Frame→Packet→Segment→Message

TCP/IP 4계층

링크→인터넷→전송→응용. 세션·표현층이 응용층에 통합됨. 실무 표준.

4쌍의 주소

이름(응용) / 포트(전송) / IP(네트워크) / MAC(링크) — 계층별 다른 주소 사용

계층 전달 범위 주소 PDU 주요 장치
응용층 (7) 응용 프로그램 이름 Message PC, 서버
전송층 (4) 프로세스 대 프로세스 포트 번호 Segment PC, 서버
네트워크층 (3) 발신지-목적지 (end-to-end) IP 주소 Packet 라우터
데이터링크층 (2) 노드 대 노드 (hop-to-hop) MAC 주소 Frame 스위치
물리층 (1) 비트 전송 - Bit 케이블, 허브
⭐ 시험 포인트
  • 물리층: 비트 스트림 → 전자기 신호 변환, 기계·전기적 특성
  • 데이터링크층: 노드-대-노드 전달, MAC 주소, 프레임 단위, 트레일러로 오류 제어
  • 네트워크층: 발신지-목적지 전달, IP 주소, 라우팅
  • 전송층: 프로세스-대-프로세스 전달, 포트 번호, 순서 번호, 세그먼트
  • TCP/IP: 세션층·표현층 없음 → 4계층
  • Switch: 2계층(MAC), 저가, 소규모, 사용자 연결
  • Router: 3계층(IP), 고가, 중대형, 외부망 연결
