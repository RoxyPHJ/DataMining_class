01

데이터 통신이란?

핵심 정의
  • 전기통신 먼 거리에서 행해지는 통신. tele는 그리스어로 "멀다"는 의미.
  • 데이터 송신자와 수신자 간에 합의된 형태로 표현된 정보. 특히 0과 1로 표현된 것.
  • 데이터 통신 전선과 같은 통신매체를 통하여 두 장치 간에 데이터를 교환하는 것.
3가지 주요 토픽
구성요소 (Components)
데이터 표현 (Data Representation)
데이터 흐름 (Data Flow)

프로토콜

04

데이터 흐름

3가지 통신 방식
Simplex
단방향 방식
한쪽 방향으로만 통신 가능. 한 지국은 송신만, 다른 지국은 수신만 가능.
예: 자판(키보드), 모니터
Half-Duplex
반이중 방식
각 지국은 송·수신 모두 가능. 단, 각 지국 간 동시 송신은 불가.
예: 워키토키, CB radio
Full-Duplex
전이중 방식
양쪽 지국 간 동시에 송·수신 가능. 신호는 링크 용량을 공유하여 양방향 전달.
예: 전화
05

네트워크 개요

네트워크 정의 및 평가기준

전송 매체 링크로 서로 연결된 장치(노드: 컴퓨터나 통신장치)의 모임

평가 기준설명주요 지표
성능 (Performance) 네트워크가 얼마나 잘 동작하는가 전달시간, 응답시간, 처리율(throughput), 지연(delay)
신뢰성 (Reliability) 네트워크가 얼마나 안정적인가 고장 빈도, 링크 복구 소요 시간
보안 (Security) 네트워크가 얼마나 안전한가 불법 접속, 바이러스로부터 보호
06

연결 유형 (Type of Connection)

점-대-점 vs 다중점
Point-to-Point

채널의 전체 용량이 두 기기 간 전송을 위해 전용으로 사용. 케이블, 전선, 극초단파, 인공위성 연결을 통해 구성.

Multipoint (Multidrop)

3개 이상의 특정 기기가 하나의 링크를 공유. 채널 용량을 공간적 또는 시간적으로 공유.

07

접속 형태 (Topology)

물리적 혹은 논리적인 네트워크 배치 방식

🔗 그물형 (Mesh)
  • 링크 수n(n-1)/2 개 (전이중 모드)
  • 포트 수각 장치마다 n-1 개의 I/O 포트
PROS
  • 전용 링크로 안정적 전송
  • 높은 신뢰성
  • 보안 우수
CONS
  • 케이블·포트 수 多
  • 설치·재구성 어려움
⭐ 스타형 (Star)
  • 채널 수1개 채널, 1개 I/O 포트
  • 중앙장치허브(Hub) — 모든 통신 중계
PROS
  • 그물형보다 저비용
  • 설치·재구성 용이
CONS
  • 허브 고장 → 전체 마비
📡 버스형 (Bus)
  • 구조다중점 형태. 탭+유도선으로 연결.
PROS
  • 설치 쉬움
  • 케이블 최소 사용
CONS
  • 재구성·결함분리 어려움
  • 중추 케이블 결함 시 전체 영향
🔄 링형 (Ring)
  • 구조양쪽 장치와 점대점 링크. 각 장치에 중계기 포함.
  • 신호항상 링을 순환
PROS
  • 설치·재구성 쉬움
  • 경보 기능 사용 가능
CONS
  • 단방향 시 링 결함 → 전체 마비
  • 해결: 이중 링 또는 결함지점 스위치
혼합형 (Hybrid)

위의 여러 접속 형태를 혼합한 구성. 실제 대부분의 네트워크가 혼합형을 사용.

08

네트워크 유형

LAN vs WAN 비교
구분LAN (근거리 통신망)WAN (광역 통신망)
범위 사무실, 건물, 학교 등 단일 구역 국가, 대륙, 전세계 — 거리 제한 없음
소유 개인소유 또는 단일 기관 통신회사가 임대 목적으로 운영
사용 목적 자원 공유 데이터, 음성, 영상, 비디오의 장거리 전송
토폴로지 버스형, 링형, 스타형 사용 점-대-점(P2P) WAN, 교환형(Switched) WAN
인터네트워크 (Internetwork / Internet)

두 개 이상의 네트워크가 서로 연결되면 "Internetwork" 또는 "internet"

09

교환 (Switching)

회선교환 vs 패킷교환
Circuit-Switched
회선교환망

종단 시스템 간에 회선이라는 전용선 이용. 통신 전 물리적 경로를 미리 설정.

Packet-Switched
패킷교환망

종단 간 통신이 패킷이라는 데이터 블록에 의해 이루어짐. 각 패킷이 독립적으로 라우팅.

10

인터넷 구조

인터넷 계층 구조

수 천 개의 상호 연결된 네트워크로 이루어짐. 백본, 제공자 네트워크, 소비자 네트워크로 구성.

11

인터넷 역사

주요 연표
~1960 이전
전신과 전화망 — 초기 원거리 통신
1961
패킷 교환망 탄생 — MIT의 클라이녹(Kleinrock)이 발표
1967
ARPANET 아이디어 제안 — ARPA가 ACM 모임에서 각 호스트를 IMP(Interface Message Processor)에 연결하는 방안 제시
1969
ARPANET 구성 — 4개 노드(UCLA, UCSB, SRI, UU) 연결. NCP(Network Computer Protocol) 소프트웨어로 호스트 간 통신
1972
Vint Cerf와 Bob Kahn이 네트워크를 통해 패킷을 전송하는 중계 하드웨어인 게이트웨이(Gateway) 개발
1973
종단-대-종단 패킷 전달 프로토콜 제안 → TCP/IP로 발전
1977
인터넷 시연 (ARPANET, 패킷 라디오, 패킷 위성)
1981
UNIX에 TCP/IP 포함 배포. CSNET 구성(NSF 지원, ARPANET 미접속 대학용)
1983
TCP/IP가 ARPANET 공식 프로토콜 채택. MILNET(군사용)과 ARPANET 분리
1980년대
NSFNET — 5대 슈퍼컴퓨터를 T1 라인(1.544Mbps)으로 연결. ANSNET — IBM, Merit, MCI의 고속 백본
1990년대~
WWW — CERN의 팀 버너스리(Tim Berners-Lee)가 개발. 멀티미디어(VoIP, Video over IP), P2P 네트워크(블록체인) 등 발전
네트워크 발전 단계

핵심 요약

Chapter 1 핵심 정리
데이터 통신 5요소
송신자, 수신자, 메시지, 전송매체, 프로토콜
데이터 흐름 3방식
단방향(Simplex), 반이중(Half-duplex), 전이중(Full-duplex)

4가지 Topology
Mesh(그물형), Star(스타형), Bus(버스형), Ring(링형)
네트워크 유형
LAN(근거리) vs WAN(광역). 복수 연결 → Internetwork
교환 방식
회선교환(전용 회선) vs 패킷교환(패킷 블록 독립 라우팅)
인터넷 계층
백본(Sprint 등) → ISP(제공자) → 소비자 네트워크
인터넷 탄생
ARPANET(1969) → TCP/IP(1973) → WWW(1991)
